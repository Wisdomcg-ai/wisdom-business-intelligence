Stage 1 — Surface

The 5-Layer Problem Unpacking Model

Most people describe the symptom and stop. The breakthrough always happens at Layer 3 — the constraint. Work through these layers before you open Claude, or use them to guide your conversation.

1
The Symptom — what you notice
The thing that annoys you or shows up as a problem. It's real, but it's not the problem — it's the signal that a problem exists underneath.
Examples
✗ "We're always chasing invoices"
✗ "The team never updates the job tracker"
✗ "I spend all my time in the business, not on it"
2
The Process — what's actually happening
Map out the steps. Who does what, when, using which tools. You're looking for where manual effort, handoffs, or gaps exist. Write it as a sequence, not a description.
Prompt to use
"Walk me through exactly what happens, step by step, from the moment [trigger] until [outcome]. Who does each step and what tool do they use?"
3
The Constraint — where it breaks ⭐ This is the one
The specific point in the process where it fails, slows down, or depends on one person. This is what you're actually solving. If you can name this precisely, the solution designs itself.
Bad → Good
✗ "Our invoicing is slow"
↓ dig deeper
✓ "Invoices only get sent when I personally approve them — there's no trigger for the admin to send without my sign-off, so they pile up when I'm on site"
4
The Cost — what not solving it actually costs
Be specific. Vague cost = low urgency = no action. Quantify in time, money, or risk. If you can't put a number on it, the problem probably isn't ready to solve yet.
Frames that work
⏱ "This costs us X hours per week"
💰 "This creates roughly $X in lost cash flow per month"
⚠️ "If we don't fix this, we risk [specific consequence]"
🔓 "Solving this unlocks [specific opportunity] we can't pursue now"
5
The Owner — why it hasn't been solved
Who owns this problem and why does it persist? Usually: no clear owner, the owner is too busy, or the solution requires technical skill nobody has. Understanding this shapes what solution will actually stick.
Questions to answer
→ Whose job is it to fix this?
→ What's stopped them?
→ What does the solution need to look like to actually get used?
The full Claude prompt for Stage 1
"I want to describe a problem in my business. I'm not looking for solutions yet — I want you to help me understand the problem more clearly. Ask me questions one at a time until you can restate the problem back to me in 2 sentences that I'd agree are exactly right."

All 4 Stages

Build Prompts — Copy and Paste

Use these exactly as written, then adapt with your own details in the square brackets. Each one is designed to move you to the next stage.

1
Surface — Get the real problem out
15 min
Stage 1 Prompt
I want to describe a problem in my business. I'm not looking for solutions yet — I want you to help me understand the problem more clearly.

Ask me questions one at a time until you can restate my problem back to me in 2 sentences that I'd agree are exactly right.

Start with: what is the problem, and who experiences it most directly?
2
Design — Build the solution architecture
20 min
Stage 2 Prompt
Based on everything we've discussed, design a solution architecture for this problem.

Give me:
1. The core components — what are the distinct parts?
2. What to build first and why — sequence matters
3. What success looks like in 90 days — specific and measurable
4. The biggest risks I should know about — be honest

Then I want to stress-test it. After you give me the architecture, I'll ask: what are you assuming about my business that might not be true?
3
Build — Make the first component real
40 min
Stage 3 Prompt
Take the first component we identified. Build it completely — give me the working version, not a description of what it would do.

If it's a document or process: write it fully, formatted and ready to use.
If it's a tool or app: write the complete code I can run or deploy.
If it's a Claude Project: write the full system prompt and instructions.
If it's a spec: write the complete technical brief a developer could build from.

Don't summarise. Build it.
4
Capture — Lock in the outcome
15 min
Stage 4 Prompt
Summarise what we built in this conversation in 3 sentences.

Then give me:
- The one thing I need to do in the next 7 days to move this forward
- The one dependency that could block me
- One question I should ask myself before I start
Stress-test prompts — use these any time in Stage 2 or 3
"What are you assuming about my business that might not be true?"
"If I could only build one component first — which one and why?"
"What would make this architecture fail?"
"What have I not asked you that I should have?"

Key Tools — Links and First Steps

You don't need all of these. You need the ones that fit your architecture. Start with the first step listed — most of these are free to begin.

Included in Pro
Claude's coding agent. Describe what you want to build, and it writes, tests, and iterates on the code. No coding knowledge required to direct it.

Free

Supabase
Free tier
A real database for your custom tools. If your solution needs to store data — client records, job tracking, certificate expiries — Supabase handles the backend. Claude Code integrates with it directly.
supabase.com — First step: create a project, Claude Code does the rest ↗
Vercel
Free tier
Deploys your web tools to a live URL in minutes. Connect your GitHub repo, push your code, and Vercel publishes it automatically. No server management.
vercel.com — First step: connect your GitHub account ↗
Zapier
Free tier
Connects your existing tools automatically. When something happens in one app, Zapier triggers an action in another. Best for automations between tools you already use.
zapier.com — Free: 5 Zaps, 100 tasks/month ↗
Make
Free tier
More powerful than Zapier for complex multi-step automations. Visual workflow builder. Better for building things that need logic, branching, and error handling.

What's Coming

Autonomous Agents — Manus & OpenClaw

These tools represent the next level — AI that works autonomously while you sleep. They're powerful, but have real caveats. Here's an honest read on both so you know what you're getting into.

Manus

Cloud-hosted No setup required Free tier available

Manus is a cloud-based autonomous agent. You give it a high-level goal — "research my top 5 competitors and build me a comparison report" — and it plans the steps, browses the web, writes code if needed, and delivers a finished file. You don't watch it work; you come back to results. Think of it as a digital contractor you brief once.

What it's good at
  • Research tasks — it browses the web and synthesises
  • One-off projects — competitive analysis, market research
  • Generating presentations, reports, and structured docs
  • No installation — works in a browser, runs in the cloud
  • Free tier to test before committing
Honest caveats
  • Credit-based pricing — costs are unpredictable per task
  • Output quality is variable — impressive when it works, rough when it doesn't
  • Not for ongoing, recurring workflows — use Claude Projects for that
  • You can't fully control how it works — it's a black box
Is this for you? Yes, if you want to try autonomous AI with zero setup. Start with a research task — "analyse my industry's top 5 competitors" — and see what comes back. Pricing: free tier, Pro from ~$20/month. Visit manus.im

OpenClaw

Self-hosted Open source Technical setup required

OpenClaw is a self-hosted AI agent that runs on your own machine and lives inside your chat apps — WhatsApp, Telegram, Slack, or Discord. You text it a command and it executes: managing your calendar, triaging emails, running scripts, browsing the web. Unlike Manus, it's always on, remembers everything across sessions, and you control it completely. One developer used it to negotiate $4,200 off a car purchase while he slept.

What makes it special
  • Runs persistently in the background — always working
  • Lives in apps you already use (WhatsApp, Slack etc.)
  • Persistent memory — it knows your context from yesterday
  • Your data stays on your hardware — not a vendor's cloud
  • Free — you only pay for the AI model API calls
  • 700+ community-built skills (Gmail, GitHub, calendars etc.)
Honest caveats
  • Requires technical setup — command line, Node.js
  • Needs a always-on machine (Mac Mini works well)
  • Requires root-level system access — real security risk if misconfigured
  • One maintainer has warned: "if you can't run a command line, this is too dangerous to use safely"
  • Still maturing — community project, not enterprise software
Is this for you? Only if you're comfortable with technical setup or have a developer who can help. This is the frontier. The payoff is significant — a persistent AI that lives in your phone and works autonomously. The risk is real if you don't know what you're doing. Visit openclaw.ai and read the security docs before installing.
