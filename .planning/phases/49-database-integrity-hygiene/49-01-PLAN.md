<objective>
Land the smallest, lowest-risk PR in Phase 49 first to build momentum:

1. **DB-01** — add nullable `deleted_at` (timestamptz) + `deleted_by` (uuid → auth.users SET NULL) to the 8 most-mutated financial tables: `financial_forecasts`, `forecast_employees`, `forecast_pl_lines`, `monthly_actuals`, `xero_pl_lines`, `cfo_report_status`, `cfo_email_log`, `account_mappings`. Add the matching `WHERE deleted_at IS NULL` partial index per table (per the `businesses.idx_businesses_deleted_at` convention from baseline:6973).
2. **DB-02** — same migration adds nullable `created_by` + `updated_by` to those 8 tables, then runs best-effort backfill: `created_by = user_id` (financial_forecasts), `created_by = mapped_by` (account_mappings, where non-null), `created_by = triggered_by` (cfo_email_log, where non-null), and a `forecast_audit_log` join for `forecast_employees` + `forecast_pl_lines` (likely a no-op because the audit-log triggers were never wired with CREATE TRIGGER — the migration COMMENT records this).
3. **DB-05** — rename the two date-only migration filenames via `git mv` to the 14-digit form, narrow the CI migration-check regex to require 14 digits going forward, and document the manual `UPDATE supabase_migrations.schema_migrations` SQL the operator must run on production after deploy so production does not try to re-apply the renamed files.

Why one PR: same 8 tables, same `ALTER TABLE` pattern, single sign-off. DB-05 is a 2-file `git mv` + a regex narrow — it rides along on the same PR with no additional risk.

**Important caveat to surface in the migration COMMENT:** RESEARCH.md confirmed `log_forecast_change` and `audit_employee_changes` exist as functions in baseline_schema.sql but are NEVER wired with CREATE TRIGGER (`grep -c "CREATE TRIGGER" baseline_schema.sql` = 0). The DB-02 backfill from `forecast_audit_log` is therefore best-effort — `forecast_employees` and `forecast_pl_lines` will likely have most `created_by` rows remain NULL because the audit log was only ever populated by app code, not DB triggers.

Output: 1 new migration file + 2 file renames + 1 CI workflow tightening + 3 vitest files + clean tsc/build.
