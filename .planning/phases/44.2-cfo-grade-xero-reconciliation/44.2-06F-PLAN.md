<objective>
Production rollout protocol. After 06A-06E ship behind PRs, this plan delivers the runbook for actually
running the migration sequence against production Supabase + production Xero data, with rollback steps
at every level.

Key principles:
- Per-tenant canary: rollout starts on a small AU/AUD tenant (JDS), then a medium AU tenant (Envisage),
  then full rollout including IICT (multi-currency). At each canary stage, run verify-production-migration.ts
  and confirm 5 gates pass before promoting to next tier.
- Reversible at every step: each migration has a rollback recipe; the orchestrator changes are
  feature-flag-gated so they can be reverted via env var without redeploy.
- Independent verifiability: the verify script runs the SAME gates as 06E's test suite but against
  live data, so a green test suite + green verify script = "production matches the reference fixtures".
- Operator-grade documentation: written so a non-developer following the runbook with someone monitoring
  could execute the production cutover safely.

Output: 2 docs + 1 verification script. No new application code or migrations.

This is the LAST plan in the rebuild block. After 06F is complete, the existing 44.2-07 through 44.2-12
(data_quality UI, daily recon cron, UAT) execute on top of a verified production foundation.
