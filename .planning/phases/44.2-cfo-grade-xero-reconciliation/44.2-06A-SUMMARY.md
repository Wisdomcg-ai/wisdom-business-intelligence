## Deviations from plan (4)

### 1. `notes` column folded into migration 000001
**Plan said:** `notes` already on main (from absorbed 44.2-06 absorber migration).
**Reality:** That migration was never merged after Path A pivot.
**Fix:** Migration 000001 added `notes text NULL` alongside `account_id` + `basis`.
**Impact:** Zero — column was always intended; just landed earlier.

### 2. Manual SQL Editor application instead of auto-deploy
**Plan said:** Supabase auto-applies migrations on PR merge.
**Reality:** Supabase Preview branches get migrations on PR open, but production main does NOT auto-apply. Confirmed in this run + suspected for prior PRs (#23 sync_jobs, etc.).
**Fix:** Operator pasted migration SQL into production Supabase Studio SQL Editor in sequence.
**Impact:** Audit trail is the merged PR + a manual copy. Captured as architectural debt — needs a separate phase to fix Supabase deploy infrastructure.

### 3. Backfill script needed 4 patches uncovered by live run

| Issue | Fix | Reason |
|-------|-----|--------|
| Script used `.eq('business_id', connection.business_id)` | Replaced with `resolveBusinessIds()` + `.in('business_id', ids.all)` | xero_connections.business_id holds businesses.id (legacy); xero_pl_lines.business_id holds profile_id |
| PostgREST 1000-row cap silently truncated | Added `.range()` pagination loop with PAGE_SIZE=1000 | Efficient Living has 1839 rows |
| `is_active=true` filter missed Envisage's inactive connection | Removed filter | 968 Envisage rows weren't reachable |
| 671 orphan rows with empty tenant_id, no xero_connections | Added end-of-main "orphan sweep" generating uuid-v5 SYNTH-AIDs | Pre-multi-tenant data |

### 4. 3,658 rows of pre-existing dual-ID drift required manual SQL backfill
**Found during pre-flight of migration 000002.** Four businesses (Efficient Living, Dragon Roofing, Envisage, IICT Group) had xero_pl_lines.business_id = businesses.id instead of profile_id. Resolved in-session with:
```sql
BEGIN;
UPDATE xero_pl_lines x SET business_id = bp.id
FROM business_profiles bp
WHERE bp.business_id = x.business_id
  AND x.business_id NOT IN (SELECT id FROM business_profiles);
-- Verified 0 orphans
COMMIT;
```
**3,658 rows updated.** This was historical drift, not caused by Phase 44.2.
