## Deviations from plan (5)

### 1. Catalog FK violation on first run — orchestrator passed wrong business_id
**Plan said:** xero_accounts catalog upsert uses `profileId` from resolveBusinessIds.
**Reality:** `xero_accounts.business_id` FK targets `businesses(id)` (legacy), not `business_profiles(id)`. First Path A sync errored on every tenant.
**Fix (PR #30):** orchestrator now passes `ids.bizId` (canonical businesses.id) to `refreshXeroAccountsCatalog`. xero_pl_lines continues to use `profileId` (its FK target since 06A). Documented in code comment as the dual-ID drift surface.

### 2. Integration page sync was destructive after 06A
**Plan said:** orchestrator handles `/api/Xero/sync-forecast`. The integration page is on a different route.
**Reality:** `/api/monthly-report/sync-xero` (the integration page button) was an old route writing wide-format `monthly_values` JSONB with wrong `business_id` and no `account_id`. After 06A's `account_id NOT NULL` constraint, every "Sync" click WIPED `xero_pl_lines` for the tenant and failed to re-insert. User clicked sync, sync said "success", table was empty.
**Fix (PR #30):** `/api/monthly-report/sync-xero` route now delegates P&L work to `syncBusinessXeroPL` (BS portion preserved until 06D). `maxDuration` bumped 60→300 to fit the orchestrator's per-month fetch loop.

### 3. Layout-dependent classification dropped 38% of OpEx for tenants with custom report layouts
**Plan said:** D-44.2-14 fix in `pl-single-period-parser` handles non-classifying sub-section titles (e.g. "Software Development" under "Less Cost of Sales").
**Reality:** That fix only handled NESTED sub-sections. JDS uses a Xero custom report layout where sub-headers are FLAT siblings of their classifying parent (e.g. "Less Operating Expenses" / "Admin Expenses" / "Advertising & Marketing" / "Office Expenses" all at top level). "Advertising & Marketing" doesn't classify → all rows dropped → $176k of OpEx invisible (Trade Show $90k, Marketing $64k, Travel $21k, Insurance $35k, etc.) → JDS YTD-Mar showed $441,093 instead of $237,410.
**Fix (PR #31):** Two-layer classifier:
- **Primary:** `classifyByXeroType()` maps `xero_accounts.xero_type` (chart-of-accounts truth) to bucket. Layout-independent. Validated against 1,154 accounts across all 18 tenants.
- **Fallback:** parser forward-carry — when a top-level Section title doesn't classify, inherit from the most recent classifying sibling. Catches FXGROUPID and SYNTH-AID rows (no catalog entry by design).

### 4. Forecast wizard cache invalidation gate too narrow
**Plan said:** Display layer is out of 06B scope.
**Reality:** Once `xero_pl_lines` had accurate other_income, the wizard's prior-year display still showed -$651 instead of $0. Root cause in `ForecastWizardV4.tsx`: localStorage-restore path gated `priorYear` refresh behind "revenue changed by more than $1". For tenants whose Xero revenue total didn't move pre/post Path A but downstream fields (other_income, opex composition) did, the fresh API response was discarded and stale cached priorYear was kept.
**Fix (PR #33):** `setPriorYear()` now runs unconditionally on mount. `initializeFromXero()` (which can clobber user-customized line splits) stays gated to actual changes. Read-only display data updates always; user-input data stays guarded.

### 5. Forecast wizard's net profit formula omitted +otherIncome
**Plan said:** Display layer out of scope.
**Reality:** Surfaced by Path A. The wizard's `calculateYearSummary` formula was `grossProfit - teamCosts - opex - depreciation - userOtherExpenses - investments` with NO `+ otherIncome` term. Long-standing bug invisible until other_income was actually populated.
**Fix (PR #32):** Added `+xeroOtherIncome - xeroOtherExpense` to the formula. Excel/PDF exports adopted shared `netProfitFromBuckets()` helper. Step8Review's what-if calc + display rows updated. Legacy `financial_metrics.net_profit_month` writes marked deprecated dead-write (no consumer reads them).
