<objective>
Balance Sheet schema — the second half of "match Xero to the cent" requires a place to store BS data
keyed by point-in-time, just as 06A did for P&L (keyed by period). This plan creates xero_bs_lines and
its wide-compat view, with the same canonical-identity discipline (account_id GUID first-class), basis
stamping, and FK enforcement that 06A established.

Difference from xero_pl_lines (key insight per Calxa-via-Cowork research): Balance Sheet is point-in-time
("Net Assets at 2026-04-30"), not range-based ("revenue Jul-Apr 2026"). So the time key is a single
balance_date, not a period_month. Mathematically: PL[period] = revenue - expenses for that range;
BS[date] = assets - (liabilities + equity) at that instant.

Per Calxa-via-Cowork Q1: Reports/BalanceSheet has the same documented periods-parameter date-arithmetic
bug as Reports/ProfitAndLoss. We avoid it the same way: single-period queries only, one per month-end.
06D will implement the sync orchestrator for BS using this schema.

Output: 2 idempotent migrations + 1 verification test suite. No code changes — schema only.
</objective>

    -- Phase 44.2 Plan 06C — Balance Sheet line storage.
    --
    -- Mirrors xero_pl_lines structure (post-06A) with point-in-time semantics:
    --   period_month date (P&L) → balance_date date (BS)
    --   amount       numeric (P&L) → balance numeric (BS)
    --
    -- Same canonical-identity discipline: account_id GUID first-class, account_code stores user-facing
    -- Xero Code, basis stamped, business_id FK to business_profiles, RLS aligned with P&L.
    --
    -- account_type is BS-specific: ('asset'|'liability'|'equity'). 3 top-level types vs P&L's 5.
    -- Sub-classifications (bank, current_asset, retained_earnings, etc.) stay in xero_accounts.xero_class.

    CREATE TABLE IF NOT EXISTS xero_bs_lines (
      id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
      business_id   uuid NOT NULL,
      tenant_id     text NOT NULL DEFAULT '',
      account_id    uuid NOT NULL,
      account_code  text NULL,
      account_name  text NOT NULL,
      account_type  text NOT NULL CHECK (account_type IN ('asset','liability','equity')),
      section       text NULL,
      balance_date  date NOT NULL,
      balance       numeric(18,2) NOT NULL DEFAULT 0,
      basis         text NOT NULL DEFAULT 'accruals' CHECK (basis IN ('accruals','cash')),
      source        text NOT NULL DEFAULT 'xero',
      notes         text NULL,
      created_at    timestamptz NOT NULL DEFAULT now(),
      updated_at    timestamptz NOT NULL DEFAULT now()
    );

    -- FK to business_profiles, mirroring 06A's discipline. Pre-flight assertion not needed
    -- because the table is newly created (no existing rows to validate).
    DO $$
    BEGIN
      IF NOT EXISTS (
        SELECT 1 FROM pg_constraint WHERE conname = 'xero_bs_lines_business_id_fk'
      ) THEN
        ALTER TABLE xero_bs_lines
          ADD CONSTRAINT xero_bs_lines_business_id_fk
            FOREIGN KEY (business_id)
            REFERENCES business_profiles(id)
            ON DELETE RESTRICT;
      END IF;
    END $$;

    -- Natural key: (business_id, tenant_id, account_id, balance_date).
    DO $$
    BEGIN
      IF NOT EXISTS (
        SELECT 1 FROM pg_constraint WHERE conname = 'xero_bs_lines_natural_key_uniq'
      ) THEN
        ALTER TABLE xero_bs_lines
          ADD CONSTRAINT xero_bs_lines_natural_key_uniq
            UNIQUE (business_id, tenant_id, account_id, balance_date);
      END IF;
    END $$;

    -- Indexes for common query patterns.
    CREATE INDEX IF NOT EXISTS xero_bs_lines_business_idx ON xero_bs_lines (business_id);
    CREATE INDEX IF NOT EXISTS xero_bs_lines_balance_date_idx ON xero_bs_lines (balance_date);
    CREATE INDEX IF NOT EXISTS xero_bs_lines_account_id_idx ON xero_bs_lines (account_id);

    -- RLS — mirror xero_pl_lines policies.
    ALTER TABLE xero_bs_lines ENABLE ROW LEVEL SECURITY;

    DROP POLICY IF EXISTS xero_bs_lines_access ON xero_bs_lines;
    CREATE POLICY xero_bs_lines_access ON xero_bs_lines
      FOR ALL
      TO authenticated
      USING (
        (SELECT role FROM system_roles WHERE user_id = auth.uid() LIMIT 1) = 'super_admin'
        OR business_id IN (SELECT auth_get_accessible_business_ids())
      );

    DROP POLICY IF EXISTS xero_bs_lines_service_role ON xero_bs_lines;
    CREATE POLICY xero_bs_lines_service_role ON xero_bs_lines
      FOR ALL
      TO service_role
      USING (true)
      WITH CHECK (true);

    COMMENT ON TABLE xero_bs_lines IS
      'Phase 44.2 06C — Balance Sheet line storage. Point-in-time (balance_date), Xero AccountID-keyed, basis-stamped, FK-protected. Mirrors xero_pl_lines design.';
    COMMENT ON COLUMN xero_bs_lines.account_type IS
      'BS top-level classification: asset | liability | equity. Sub-classifications live in xero_accounts.xero_class.';
    COMMENT ON COLUMN xero_bs_lines.balance_date IS
      'Point-in-time as-of date for this balance. Typically month-end (e.g. 2026-04-30). Single-period BS queries only — no aggregation parameters.';
    ```

    Naming follows 06A pattern (20260430000010_ to leave numeric room between 06A's 000001-000004
    and 06C's BS-related migrations).

    -- Phase 44.2 Plan 06C.2 — Balance Sheet wide-compat view.
    --
    -- Mirrors xero_pl_lines_wide_compat: aggregates per-account into a balances_by_date jsonb column,
    -- exposing both account_id (canonical) and account_code (informational), plus basis stamp.

    CREATE OR REPLACE VIEW xero_bs_lines_wide_compat AS
    SELECT
      business_id,
      tenant_id,
      account_id,
      account_code,
      account_name,
      account_type,
      section,
      basis,
      jsonb_object_agg(to_char(balance_date, 'YYYY-MM-DD'), balance) AS balances_by_date,
      min(created_at) AS created_at,
      max(updated_at) AS updated_at
    FROM xero_bs_lines
    GROUP BY
      business_id,
      tenant_id,
      account_id,
      account_code,
      account_name,
      account_type,
      section,
      basis;

    ALTER VIEW xero_bs_lines_wide_compat SET (security_invoker = on);

    COMMENT ON VIEW xero_bs_lines_wide_compat IS
      'Phase 44.2 06C.2 — wide-format BS compat view. balances_by_date jsonb keyed by YYYY-MM-DD (point-in-time, NOT period range). security_invoker honored.';
    ```
