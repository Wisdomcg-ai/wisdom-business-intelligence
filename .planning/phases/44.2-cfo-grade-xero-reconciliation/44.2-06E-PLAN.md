<objective>
The verification gate. Five reconciliation invariants from Calxa-via-Cowork Q3, expressed as automated
tests that run on every commit, parameterized over 3 reference tenants × 3 month-ends. 45 individual
assertions per CI run.

Per the Calxa-via-Cowork research, these 5 gates jointly express "your monthly reporting matches Xero
to the cent":
  Gate 1 — Single-period vs aggregated PL: oracle agreement, catches by-month bug regressions
  Gate 2 — PL ↔ BS articulation: catches sync-time errors that break the fundamental accounting equation
  Gate 3 — TB balanced: catches missing journal lines (Σ debits == Σ credits is universal)
  Gate 4 — BS in balance: Net Assets == Equity is THE accounting fundamental
  Gate 5 — Web parity: manual cross-check against Xero web PDF for tenant-supplied evidence

This is the final assurance layer. After 06E is green for all 45 assertions, we know:
- Path A's per-month single-period query is structurally correct (gate 1)
- P&L and BS are internally consistent (gate 2)
- No journal lines were dropped or duplicated (gate 3)
- BS arithmetic is sound (gate 4)
- Our numbers tell the same story coaches see in Xero (gate 5)

If any of these fail in CI later, we know exactly which class of bug was introduced.

Output: 2 capture scripts + 12 fixtures + 1 integration test file + 1 evidence doc. Tests run in CI on every commit. Fixtures are static input — re-capture only when intentionally re-baselining (rare; documented).

This plan does NOT change production code. It is a verification harness.

Tenants: JDS (small AU/July, 1 tenant), Envisage (mid AU/July, 1 tenant), IICT (multi-currency
consolidation, 4+ tenants including HKD subsidiary). Three tenants × three month-ends gives broad
coverage of fiscal-period boundaries (Feb=28 days, Mar=31, Apr=30).
</objective>
