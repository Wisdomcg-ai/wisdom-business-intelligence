## Design notes

- Banner is **non-blocking** per CONTEXT.md UX decision — sets `data-integrity="degraded"` attribute on its root for future opacity-overlay opt-in by surrounding content. The 4 full-banner mounts could later add `className={dataQuality !== 'verified' ? 'opacity-70' : ''}` to their immediately-following content container; not done in this PR to keep visual changes minimal pending UX review.
- Coach dashboard variant: compact strip linking to `/coach/clients`. The full per-tenant detail lives at the per-client page (each affected client renders its own full DataIntegrityBanner). This avoids duplicating drawer logic for a multi-business shape that the existing drawer doesn't model.
- `useMonthlyReport` hook now surfaces `dataQuality` + `perTenantQuality`. Consolidated path doesn't yet populate these; non-consolidated path (single business) does.
