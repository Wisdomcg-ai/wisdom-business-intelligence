## Two real Xero quirks discovered along the way

### Finding A — JDS FY-total OPEX rollup

For JDS FY26, summing 11 single-period monthly PL queries gives:
- Revenue: $6,059,382.99 — exactly matches FY-total query ✅
- COGS: $2,747,169.08 — exactly matches FY-total query ✅
- **OPEX: $2,649,714.45 — FY-total reports $2,656,182.42 ($6,467.97 higher)** ❌

Xero rolls up some OPEX account differently when the query range spans the full FY vs single months. Path A monthly was verified to the cent against Xero web PDF in 06B (Sales-Hardware $237,409.92), so the per-month captures are the source of truth. The single-period FY-total query is the buggy oracle in this comparison.

**Implication for Gate 1:** the existing "Σ(monthly) == FY-total" formulation cannot pass on a tenant with this quirk. Two paths forward:
- **(a) Per-account comparison.** Surface specific accounts where monthly_sum ≠ FY-total — tenants drift on a small number of accounts only, those become the actionable failure message.
- **(b) Production xero_pl_lines as source-of-truth.** Use the actual Path A writes as the authoritative monthly sum and only compare to Xero web PDF (Gate 5 manual evidence).

Marked Gate 1 `it.todo` with the diagnostic embedded; assertion logic stays in code for the next iteration to fill in.

### Finding B — IICT-HK by-month query is destructively broken

For IICT-HK Mar 2026:
- Single-period query: revenue $1,736,150 / opex $303,664 / **net profit $1,432,485**
- by-month query (one column of a `?periods=11&timeframe=MONTH` response): revenue $0 / opex $7,975 / **net profit -$7,975**

A 1000× discrepancy on the same month, same tenant, same Xero data. **This is the strongest empirical evidence yet that Path A (single-period per month) was the right architecture.** The Calxa Q1 documented Xero bug expresses itself most violently on this multi-currency tenant.

**Implication for Gate 2:** must use single-period monthly fixtures, never by-month. Updated harness in this PR (commit `0cabeac`) — Gate 2 now reads `{tenant}-pl-single-{YYYY-MM}.json` for the month under test.
