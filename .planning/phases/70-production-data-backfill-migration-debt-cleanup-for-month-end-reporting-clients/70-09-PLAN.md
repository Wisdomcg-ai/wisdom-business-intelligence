Create `scripts/70-09-C2-cron-heartbeat-check.mjs` — READ-ONLY (no --apply, no mutations).

Algorithm:

1. Env load: `config({ path: '.env.local' })` with fallback `.env`. Use `SUPABASE_SECRET_KEY` / `SUPABASE_SERVICE_KEY` fallback.

2. KNOWN_CRONS array — populate from vercel.json. Example structure:
```js
const KNOWN_CRONS = [
  { path: '/api/cron/refresh-xero-tokens',  cadence_hours: 6,  critical: true  },
  { path: '/api/cron/sync-all',             cadence_hours: 24, critical: false },
  { path: '/api/cron/reconciliation-watch', cadence_hours: 24, critical: false },
  // ... enumerate remaining from vercel.json
];
```

3. For each cron path:
   a. `sb.from('cron_heartbeats').select('ran_at, success').eq('cron_path', P).order('ran_at', { ascending: false }).limit(1)` to get last_run.
   b. Count last 24h ticks: `sb.from('cron_heartbeats').select('id', { count: 'exact', head: true }).eq('cron_path', P).gte('ran_at', NOW_MINUS_24H_ISO)`
   c. Count last 24h failures: same as (b) plus `.eq('success', false)`.
   d. `hours_since_last_run = (Date.now() - new Date(last_run).getTime()) / 3.6e6`.
   e. Classify:
      - HEALTHY: `hours_since_last_run < cadence_hours` AND `failures_last_24h === 0`
      - WARN: `cadence_hours <= hours_since_last_run < 2 * cadence_hours` OR (`ticks_last_24h > 0` AND `failures_last_24h > 0`)
      - CRITICAL: `hours_since_last_run >= 2 * cadence_hours` OR `last_run === null` (table populated but no rows for this path)
      - UNKNOWN: query error (table missing — Phase 69 migration not applied)

4. Print per-cron status to stdout, e.g.:
```
=== Cron Health Check ===
HEALTHY  /api/cron/refresh-xero-tokens   last=2h ago  cadence=6h  ticks_24h=4 failures_24h=0
WARN     /api/cron/sync-all              last=26h ago cadence=24h ticks_24h=0 failures_24h=0
CRITICAL /api/cron/reconciliation-watch  last=NEVER   cadence=24h ticks_24h=0 failures_24h=0
```

5. Exit codes:
   - ALL HEALTHY: exit 0.
   - ANY WARN, none CRITICAL: exit 0 (warn-not-block per CONTEXT).
   - Critical-flagged cron (refresh-xero-tokens) is CRITICAL: exit 0 still, but print loud final lines:
     ```
     WARNING: refresh-xero-tokens is CRITICAL.
     Phase 70 verification (70-08) reads Xero data that may be stale or unrefreshed.
     Confirm Phase 69 deployment landed in production.
     ```
   - cron_heartbeats table missing (Phase 69 migration not in this DB): exit 0 with message "cron_heartbeats not present — Phase 69 not yet in production; Phase 70's Xero-dependent verifications are unconfirmed".
