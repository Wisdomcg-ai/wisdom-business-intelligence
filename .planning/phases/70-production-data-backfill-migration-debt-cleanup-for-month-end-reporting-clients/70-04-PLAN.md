<objective>
Backfill `subscription_budgets.renewal_month` for every row where `frequency='annual'` AND `renewal_month IS NULL`. Audit found 44/44 Envisage, 47/47 JDS, 0 rows IICT (because IICT has 0 subs entirely — that's covered in 70-07).

Purpose: The renewal_month column drives any "subscription renewals next month" feature + correct annual-cost spreading in cashflow. Today every annual sub is silently un-renewable. Source data: Xero bank transactions for the vendor over the last 24 months — the most recent matching transaction's calendar month is the renewal anchor.

Output: A two-mode script. Default = dry-run preview listing every NULL renewal_month row, the matched Xero transaction (if any), the proposed renewal_month, and unresolved rows requiring manual entry. `--apply` = updates `renewal_month` for resolved rows; surfaces unresolved rows for Matt to enter manually.
</objective>

**FILE EXTENSION IS `.ts` (TypeScript) — run via `tsx` (matches the precedent at `scripts/verify-production-migration.ts`).** This is the canonical location and extension; do not create a `.mjs` variant. Vendor-normalization imports `createVendorKey` + `extractVendorName` from `src/lib/utils/vendor-normalization.ts` directly — no transpile step needed because `.ts` runs natively under tsx.

Algorithm:

1. Env + APPLY flag + skipXero flag.

2. For each `businesses.id`:
   a. Read all `subscription_budgets` rows where `business_id = bid AND frequency = 'annual' AND renewal_month IS NULL AND is_active = true`. Skip business if empty.
   b. Read `xero_connections` where `business_id = bid AND is_active = true`. If empty → can't query Xero → mark all rows as "unresolved (no Xero connection)".
   c. For each tenant, fetch the valid access token via `getValidAccessToken({ id: connection.id, business_id, tenant_id })` from `src/lib/xero/token-manager.ts`. Wrap in try/catch — 401/token-expired here is a known risk (Phase 69 cron health gate). On failure: print clear "Xero token unavailable for <tenant_name> — skipping" and mark rows as "unresolved (token error)". Do NOT crash the whole script.
   d. For each tenant with a valid token: GET `https://api.xero.com/api.xro/2.0/BankTransactions?where=Type%3D%3D%22SPEND%22%20AND%20Date%20%3E%3D%20DateTime(<2y_ago>)`. Paginate via `page=N` (Xero returns 100 per page) until empty. Cache the merged transaction list per-tenant in memory.

3. Vendor matching per row:
   - `target_key = createVendorKey(row.vendor_name)`
   - For each cached transaction: compute `tx_key = createVendorKey(tx.Contact?.Name || extractVendorName(tx.Contact?.Name, tx.Reference || ''))`.
   - Find matches: `transactions.filter(tx => createVendorKey(...) === target_key)`.
   - If matches found: pick the most recent (`tx.Date` max). Extract month: `new Date(tx.Date).getUTCMonth() + 1` → integer 1-12.
   - If no matches: mark as "unresolved (no Xero transactions for this vendor in last 24mo)".

4. Dry-run output:
```
═══════════════════════════════════════════════════════
business=Envisage (bid=8c8c63b2...)
  ✓ MATCH    vendor="Mailchimp"          → renewal_month=3   (matched tx 2026-03-12 $750.00)
  ✓ MATCH    vendor="Adobe Creative"     → renewal_month=10  (matched tx 2025-10-04 $1200.00)
  ⚠ UNRES    vendor="LinkedIn Premium"   → no Xero tx in last 24mo (needs manual entry)
  ⚠ UNRES    vendor="Slack"              → token error (Xero connection not refreshing)
  ...
  Resolved: 18 / 44   Unresolved: 26 / 44
═══════════════════════════════════════════════════════
```

5. If `--apply`:
   - For each "MATCH" row: `UPDATE subscription_budgets SET renewal_month = X, updated_at = NOW() WHERE id = <row.id>`. Per-row try/catch.
   - For "UNRES" rows: do NOT write anything. Print them clearly at the end.
   - Append the unresolved list to `.planning/phases/70-.../70-04-unresolved-renewals.json` (the seed file for Matt's manual-entry step in Task 2).

6. Summary at end:
   - Per-business: resolved count / total NULL count / unresolved count
   - Top-level: total resolved across production / total still NULL

Skip if empty or 'skip'. Wait until Matt drives this interactively in Task 2 — the auto flow only handles Xero-matched rows.
