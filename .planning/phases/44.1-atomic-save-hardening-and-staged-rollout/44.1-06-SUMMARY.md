## What was delivered

### 1. PITR-RUNBOOK.md (Task 1)

Operational rollback recipe with 8 numbered steps:

1. **Identify the target row set** — SELECT against current state.
2. **Open a PITR clone (READ-ONLY)** — Supabase dashboard → Backups → Restore from PITR → temporary DB URL.
3. **DRY-RUN — fetch what would be restored** — SELECT against the clone, verify row count + manual rows + forecast_months presence.
4. **Export the restore set as INSERT statements** — `quote_literal` driven SELECT that emits `INSERT … ON CONFLICT (id) DO UPDATE SET`.
5. **COMMIT — apply restore to production** — `BEGIN` + `pre_restore_snapshot` temp table + paste exports + verify counts + `COMMIT` (or `ROLLBACK`).
6. **Post-restore RLS check** — `SET ROLE authenticated; SELECT COUNT(*); RESET ROLE;` — confirms Wave 5 RLS hotfix `ec5055e` is intact.
7. **Bump computed_at to satisfy 44-08 freshness invariant** — `UPDATE forecast_pl_lines SET computed_at = now()` + `UPDATE financial_forecasts SET updated_at = now() - interval '1 second'` so the D-18 inequality holds.
8. **Sanity check via app** — coach reload + `cfo_report_status` reconcile.

Includes a "Rollback of the restore" section + Forensics requirements (forensic-log.md append + Sentry release-note issue).
