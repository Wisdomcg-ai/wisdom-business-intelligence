## Explicit decision text (for 44.1-02 to read)

**44.1-02 SHOULD include a null-account_code backfill prologue, based on N=236, M=0.**

The backfill statement to ship as the prologue:

```sql
UPDATE forecast_pl_lines
SET account_code = 'ACCT-MISSING-' || id::text
WHERE account_code IS NULL
  AND is_manual = false;
```

After the prologue runs, 44.1-02 can safely ship the partial unique index:

```sql

  ON forecast_pl_lines (forecast_id, account_code)

```

Because M=0, no row-level dedup is required between the prologue and the index creation.
