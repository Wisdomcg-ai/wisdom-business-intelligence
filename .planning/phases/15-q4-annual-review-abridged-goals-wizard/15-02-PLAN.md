Add a new `syncAnnualReview()` method to StrategicSyncService. Place it BEFORE the `syncAll()` method. Import `NextYearTargets` and `AnnualInitiativePlan` from `'../types'`.

The method signature:
```typescript
async syncAnnualReview(
  businessId: string,  // Must be profileBusinessId (business_profiles.id)
  userId: string,
  nextYearTargets: NextYearTargets,
  annualInitiativePlan: AnnualInitiativePlan,
  nextYear: number
): Promise<{ success: boolean; errors: string[] }>
```

Implementation (two parts):

**Part A: Roll forward 3-year financial targets (targeted update, NOT full saveFinancialGoals)**
1. Load current goals: `supabase.from('business_financial_goals').select('*').eq('business_id', businessId).maybeSingle()`
2. If no row found, try the same multi-ID fallback pattern used by `syncQuarterlyTargets` (user.id, then business_profile_id) -- reuse the same lookup sequence
3. If still no row found, log warning and continue (not a fatal error -- may be new business)
4. Build update payload with roll-forward logic:
   - `revenue_year1: nextYearTargets.revenue` (A4.3 targets become new Year 1)
   - `gross_profit_year1: nextYearTargets.grossProfit`
   - `net_profit_year1: nextYearTargets.netProfit`
   - `revenue_year2: current?.revenue_year2 || 0` (current Year 2 stays as new Year 2 baseline)
   - `gross_profit_year2: current?.gross_profit_year2 || 0`
   - `net_profit_year2: current?.net_profit_year2 || 0`
   - `revenue_year3: nextYearTargets.stretchRevenue || current?.revenue_year3 || 0` (stretch target or keep current Year 3)
   - `gross_profit_year3: nextYearTargets.stretchGrossProfit || current?.gross_profit_year3 || 0`
   - `net_profit_year3: nextYearTargets.stretchNetProfit || current?.net_profit_year3 || 0`
5. IMPORTANT: Do NOT update `revenue_current`, `gross_profit_current`, `net_profit_current` or any other columns -- targeted update only on year1/year2/year3
6. Execute: `supabase.from('business_financial_goals').update(payload).eq('id', goalsRow.id)`

**Part B: Sync next-year initiatives to strategic_initiatives**
1. Iterate `annualInitiativePlan.initiatives`
2. For each initiative, determine step_type from `initiative.quarterAssigned`:
   - Map 'q1' -> 'q1', 'q2' -> 'q2', etc. (lowercase, strip any suffix like '-2027')
   - Default to 'q1' if quarterAssigned is missing
3. Check if initiative.id is a valid UUID (existing carry-forward) vs a temp ID (new)
4. For existing UUIDs: UPDATE status, notes, step_type, quarter_assigned, and set `fiscal_year = nextYear`
5. For new/temp IDs: INSERT new row with:
   - business_id: businessId (profileBusinessId)
   - user_id: userId
   - title: initiative.title
   - category: initiative.category || 'misc'
   - step_type: resolved from quarterAssigned
   - source: 'annual_review'
   - fiscal_year: nextYear
   - status: 'not_started'
   - idea_type: 'strategic'
   - selected: true
   - assigned_to: initiative.assignedTo || null
   - notes: initiative.notes || null
6. Return `{ success: true, errors: [] }` on success, collect errors for each failure

Console log with `[StrategicSync]` prefix for all operations, consistent with existing methods.

CRITICAL PITFALL AVOIDANCE:
- Use `profileBusinessId` (business_profiles.id) not `businesses.id` -- same as syncAll pattern
- Use targeted `.update()` not `FinancialService.saveFinancialGoals()` to avoid overwriting current-year data
- Stamp `fiscal_year = nextYear` on every initiative to prevent duplicate loading in Goals Wizard
