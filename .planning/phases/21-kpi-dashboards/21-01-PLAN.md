  <action>
    Create GET /api/forecast/dashboard-actuals endpoint.

    Query params: businessId (required), fiscalYear (optional, defaults to current), yearStartMonth (optional, defaults to 7).

    Logic:
    1. Auth check: supabase.auth.getUser() — 401 if missing.
    2. Use resolveBusinessIds(supabase, businessId) to get ID candidates (dual-ID pattern — CRITICAL per project memory).
    3. For each ID candidate, query financial_forecasts table: .eq('business_id', id).eq('is_active', true).order('fiscal_year', { ascending: false }).limit(1). Break on first match.
    4. If no forecast found, return { data: null, hasData: false } with 200 (not an error — business may not have a forecast yet).
    5. Query forecast_pl_lines for that forecast_id: select account_name, category, account_type, actual_months, forecast_months, is_from_xero.
    6. Generate monthKeys via generateFiscalMonthKeys(fiscalYear, yearStartMonth).
    7. Aggregate by month: for each monthKey, sum actual_months[key] across all revenue lines (isRevenue check), all COGS lines (isCOGS check), and remaining = opex. Compute:
       - revenueActual, revenueForecast
       - cogsActual, cogsForecast
       - gpActual = revenueActual - cogsActual, gpForecast = revenueForecast - cogsForecast
       - opexActual, opexForecast
       - npActual = gpActual - opexActual, npForecast = gpForecast - opexForecast
    8. Also query financial_metrics for the same business to get last_synced_at: .select('updated_at').order('updated_at', { ascending: false }).limit(1).
    9. Return JSON: { data: { months: [{ month: "2025-07", label: "Jul", revenueActual, revenueForecast, gpActual, gpForecast, npActual, npForecast }], lastSyncedAt: string | null }, hasData: true }.

    Use the SAME isRevenue/isCOGS classification pattern as quarterly-summary/route.ts — copy those helpers. Use createRouteHandlerClient from @/lib/supabase/server. Mark route as export const dynamic = 'force-dynamic'.

    Then create useXeroActuals hook:
    - Signature: export function useXeroActuals(businessId: string | undefined)
    - Returns: { chartData: MonthlyChartPoint[] | null, lastSyncedAt: string | null, isLoading: boolean, hasData: boolean }
    - MonthlyChartPoint interface: { month: string, label: string, revenueActual: number | null, revenueForecast: number | null, gpActual: number | null, gpForecast: number | null, npActual: number | null, npForecast: number | null }
    - Fetch from /api/forecast/dashboard-actuals?businessId={id} in useEffect when businessId is truthy.
    - Do NOT fetch on every render — only when businessId changes.
    - Export MonthlyChartPoint type for use by chart component.
  </action>
