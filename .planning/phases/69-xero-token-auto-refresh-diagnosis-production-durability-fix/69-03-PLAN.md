### Sub-action H3 — Verify persistence on update
Edit `src/lib/xero/token-manager.ts` at the success-path update (lines 401-422). Current code:
```typescript
const { error: updateError } = await supabase
  .from('xero_connections')
  .update({ access_token: encrypt(tokens.access_token), refresh_token: encrypt(tokens.refresh_token), expires_at: newExpiry.toISOString(), updated_at: new Date().toISOString() })
  .eq('id', connectionId);

if (updateError) { /* logs and returns success anyway with database_error flag */ }
```
Add a row-count assertion:
```typescript
const { data: updatedRows, error: updateError } = await supabase
  .from('xero_connections')
  .update({ /* same payload */ })
  .eq('id', connectionId)
  .select('id'); // returns the updated row(s)

if (updateError || !updatedRows || updatedRows.length === 0) {
  // Fail loud — the refresh would otherwise silently disappear.
  try {
    Sentry.captureMessage('Xero token refresh persisted zero rows', {
      level: 'error',
      tags: {
        invariant: 'xero_refresh_persist_failed',
        connection_id: connectionId,
        tenant_id: ctx.tenant_id ?? 'unknown',
        business_id: ctx.business_id ?? 'unknown',
      },
      extra: { update_error: updateError?.message, updated_row_count: updatedRows?.length ?? 0 },
    } as any);
  } catch { /* swallow */ }
  return {
    success: false,
    error: 'database_error',
    message: updateError?.message ?? 'Update returned zero rows — possible RLS or id mismatch',
    shouldDeactivate: false,
  };
}
```
Also: add `count: 'exact', head: false` if the supabase-js version supports it for explicit count semantics. Test the actual mock shape returned by supabase-js in vitest first.
