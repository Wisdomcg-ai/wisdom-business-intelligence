<phantom_detection_sql>
The exact detection query (per planning context, copy verbatim to `psql` / `supabase db execute`):

```sql
-- Phantom businesses: owner is a team-member of ANOTHER business, AND this row has no activity.
SELECT
  b.id,
  b.name,
  b.business_name,
  b.owner_id,
  u.email,
  b.created_at,
  (SELECT COUNT(*) FROM xero_connections WHERE business_id = b.id) AS xero_n,
  (SELECT COUNT(*) FROM financial_forecasts WHERE business_id = b.id) AS forecast_n,
  (SELECT COUNT(*) FROM business_users WHERE business_id = b.id AND status = 'active') AS team_n
FROM businesses b
JOIN auth.users u ON u.id = b.owner_id
WHERE (b.business_name = 'My Business' OR b.business_name IS NULL OR b.name = 'My Business')
  AND NOT EXISTS (SELECT 1 FROM xero_connections   WHERE business_id = b.id)
  AND NOT EXISTS (SELECT 1 FROM financial_forecasts WHERE business_id = b.id)
  AND NOT EXISTS (SELECT 1 FROM business_users     WHERE business_id = b.id AND user_id <> b.owner_id AND status = 'active')
  AND EXISTS     (SELECT 1 FROM business_users     bu WHERE bu.user_id = b.owner_id AND bu.business_id <> b.id AND bu.status = 'active')
ORDER BY b.created_at DESC;
```

Companion query for orphan business_profiles (run separately):

```sql
-- Orphan profiles: business_id NULL, user is a team-member elsewhere.
SELECT
  bp.id,
  bp.user_id,
  bp.company_name,
  bp.business_name,
  bp.created_at,
  u.email,
  (SELECT bu.business_id FROM business_users bu WHERE bu.user_id = bp.user_id AND bu.status = 'active' LIMIT 1) AS team_business_id
FROM business_profiles bp
JOIN auth.users u ON u.id = bp.user_id
WHERE bp.business_id IS NULL
  AND EXISTS (SELECT 1 FROM business_users bu WHERE bu.user_id = bp.user_id AND bu.status = 'active')
ORDER BY bp.created_at DESC;
```
