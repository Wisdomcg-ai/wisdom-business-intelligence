# Plan 72-01 — Diagnose Step 3 Extended-Period Bug

## Objective

Read-only investigation of `Step3RevenueCOGS.tsx` (and adjacent wiring) to identify
why extended-period plans (Armstrong: `plan_start_date = 2026-06-01`,
`is_extended_period = true`, `year1_months = 13`) only render 3 months of
revenue/COGS inputs instead of 13.

Produce a named root cause + fix scope that 72-02 will follow.

## Context

- `docs/forecast-wizard-extended-period-bug.md` — draft analysis dated 2026-05-29 (pre-phase)
- Phase 68 B15 already solved the same family of issues in `quarters.ts`

- Bug symptom: Armstrong opens Step 3 on 2026-05-31 and sees 3 editable
  months (Apr/May/Jun 2026) instead of 13 months from Jun 2026 → Jun 2027.
