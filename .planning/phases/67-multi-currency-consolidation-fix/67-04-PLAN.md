# 67-04 — UI signaling for FX translation

## Goal

When the wizard shows AUD totals that were translated from another currency, the operator can see the translation context (rate, source month, tenant breakdown). Missing FX rates surface clearly with a fix path.

## Acceptance

- IICT operator sees an "AUD (translated)" badge on Step 2 / Step 3 totals
- Hovering shows: presentation currency, tenant currencies, rates used
- If any required FX rate is missing, a banner appears with a CTA link to the admin FX-rates page
- Single-tenant operators see no badge (no translation happened) — UI quiet for the 95% case

## Tasks

### Task 1 — Extend API response shape

Both `/api/Xero/pl-summary` and the `getMonthlyComposite` return value gain an optional `fx_context` field:

```ts
fx_context?: {
  presentation_currency: 'AUD'
  tenant_currencies: Record<string /* tenant_name */, string /* code */>
  rates_used: Record<string /* `${pair}::${YYYY-MM}` */, number>
  missing_rates: Array<{ pair: string; month: string; severity: 'warn' | 'block' }>
}
```

The engine already produces this; just plumb it through.

### Task 2 — `<FXBadge>` component

New: `src/components/fx/FXBadge.tsx`

- Small chip "AUD (translated)" next to consolidated totals
- Hover/tooltip lists: presentation currency, per-tenant currencies, sample rate used
- Renders nothing when there's no translation (single-tenant or all-AUD)

### Task 3 — `<MissingRatesWarning>` banner

New: `src/components/fx/MissingRatesWarning.tsx`

- Top of Step 2 / Step 3 when `fx_context.missing_rates.length > 0`
- Lists missing periods (e.g. "Missing HKD/AUD monthly average for 2024-08, 2024-09")
- CTA → `/admin/consolidation/fx-rates` (existing admin page)

### Task 4 — Wire components into wizard

- Step2PriorYear.tsx: badge near totals, banner if missing rates
- Step3RevenueCOGS.tsx: same
- Page-load card (`src/app/finances/forecast/page.tsx`): badge near YTD card

### Task 5 (optional) — Per-tenant breakdown in Step 2

If we want to surface the per-tenant contribution to the consolidated total:
- A collapsible "Tenants" detail under the prior-year card
- Each tenant row: name, currency, raw amount, translated amount, rate used

This is a UX nicety; not required to fix the bug.

**Low.** UI-only additions. No data semantics change.

## Rollback

Components default to rendering nothing when `fx_context` is absent or empty — backwards compatible with the un-extended API shape. Code can ship before the API shape change.

## Verification

- IICT: badge + correct rate displayed; missing-rate warning if any month gaps
- Single-tenant clients (Envisage, JDS, etc.): no badge, no banner — UI quiet
- Hover/tooltip accessibility: keyboard-focusable, screen-reader-friendly
