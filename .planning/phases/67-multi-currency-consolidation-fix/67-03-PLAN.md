### Task 3 — `/api/Xero/sync-forecast` (push-back policy)

**Open question for Matt before code:**

When pushing forecast lines BACK to Xero for a multi-currency business, should each tenant receive:
- (a) AUD values as-is (operator forecast is in AUD; tenant absorbs the FX)
- (b) Values translated *back* to the tenant's functional currency at the rate used in the forecast period

Recommendation: **(b)**. Forecast is denominated in AUD for presentation, but the push-target tenant's ledger is in HKD; pushing AUD into a HKD ledger is incoherent. Translate back per-tenant using the same FX rate that produced the AUD presentation.

Implementation contingent on the decision.

### Task 4 — Non-consolidated monthly-report views for multi-currency businesses

These views show **mixed-currency raw sums** which are meaningless. Options:
- **(a)** Hide the non-consolidated view in the UI for multi-currency businesses; redirect to consolidated.
- **(b)** Show the view with a prominent banner: "This view shows raw tenant currencies — use the Consolidated view for AUD totals."
- **(c)** Make the non-consolidated view itself FX-aware (route through engine).

Recommendation: **(a) + (b)** combo. Default-hide (i.e. redirect or replace the link with consolidated), but if reached directly, show banner.
