# 67-01 ‚Äî Capture & backfill xero_connections.functional_currency

## Goal

Every active Xero connection's `functional_currency` reflects the actual Xero org's `BaseCurrency`. The connection-create path and the sync path both keep it that way going forward.

## Acceptance

- Backfill script runs in dry-run mode and prints a diff of mismatches
- Applied to production, "IICT Group Limited" goes from `AUD` ‚Üí `HKD`
- Any other mis-tagged connections in production are corrected
- New connections (OAuth callback) capture `BaseCurrency` at creation
- Existing connections re-read `BaseCurrency` on each sync and update if changed

## Tasks

### Task 1 ‚Äî Backfill script

New file: `scripts/backfill-functional-currency.ts`

Behavior:
1. Load all active `xero_connections` rows
2. For each, call `getValidAccessToken` (existing helper) + `getOrganisationBaseCurrency` (likely needs new helper in `src/lib/xero/organisation.ts` if not already exposed)
3. Build a diff: `[{ connection_id, tenant_name, stored, actual }]` where stored != actual
4. Print diff. With `--apply` flag, run UPDATEs (one per row, audit log included)
5. Without `--apply`, dry-run only

Run via: `npx tsx scripts/backfill-functional-currency.ts [--apply]`

### Task 2 ‚Äî Capture BaseCurrency on new connections

Audit where new `xero_connections` rows get inserted. Likely candidates (grep):
- `src/app/api/Xero/auth/callback/route.ts` (OAuth callback)
- `src/app/api/consolidation/tenants/[connectionId]/route.ts` (admin add-tenant)

For each insert site, ensure `functional_currency` is populated from `/Organisation.BaseCurrency` at insert time. If the existing code already does this for the *primary* tenant but not for *additional* tenants on a consolidated business, fix the additional-tenants path.

### Task 3 ‚Äî Keep it fresh on each sync

In `src/lib/xero/sync-orchestrator.ts`, when iterating active connections, re-read `BaseCurrency` and update if different. Don't write on every sync ‚Äî only on diff (no-op fast path).

This guards against: org changes currency in Xero (rare) OR connection was created before this hardening landed (i.e., the backfill window).

## Files

- New: `scripts/backfill-functional-currency.ts`
- Likely modified: `src/lib/xero/organisation.ts` (expose `getOrganisationBaseCurrency` if not already)
- Modified: `src/app/api/Xero/auth/callback/route.ts` or equivalent (capture on connect)
- Modified: `src/app/api/consolidation/tenants/[connectionId]/route.ts` (capture on admin add)
- Modified: `src/lib/xero/sync-orchestrator.ts` (refresh on sync)
- No migration (column already exists)

## Risk

**Low.** Pure data + sync hardening. Wrong `functional_currency` just means the FX engine doesn't translate (status quo). Right value triggers translation when 67-02 lands.

## Rollback

Revert via SQL update from backup of `xero_connections.functional_currency` column. Code changes are pure additions ‚Äî also revertable.
