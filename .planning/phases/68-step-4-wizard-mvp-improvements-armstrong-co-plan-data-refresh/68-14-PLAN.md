INTERFACES:

Step4Props quarterlyTargets type (lines 21-22):
  quarterlyTargets: Record<string, { q1: string; q2: string; q3: string; q4: string; current_remainder?: string }>

We extend this type to also allow an optional `notes` field at the per-period level. Two options:
  Option A: Add `notes?: string` at the OUTER level (per metricKey — represents notes about the metric across quarters). NOT what we want.
  Option B: Re-shape the inner record. But that breaks every consumer.
  Option C: Add a SPECIAL metricKey 'period_notes' whose value shape is `{q1, q2, q3, q4, current_remainder}` storing the notes text. This piggybacks on the existing shape without changing the type.

We choose Option C. The metricKey is the literal string 'period_notes'. Reads: `quarterlyTargets['period_notes']?.[q.id] ?? ''`. Writes: `setQuarterlyTargets({ ...quarterlyTargets, period_notes: { ...(quarterlyTargets.period_notes || { q1:'', q2:'', q3:'', q4:'' }), [q.id]: value } })`.

This way:
- No type changes required.
- Strategic-planning-service.ts persistence reads/writes the metricKey 'period_notes' as just another row (since it already iterates all keys in quarterlyTargets).
- Tolerant read: `?? ''` makes missing notes default to empty string.
- Backward compatibility: Existing rows without 'period_notes' metricKey continue to work.

Render placement: at the BOTTOM of each quarter card's flex-col, after the existing `<div className="space-y-1.5 flex-1 min-h-[60px]">` items block, before the closing `</div>` of the quarter card wrapper.
