<objective>
Fix the Step 3 revenue-initialization bug so the wizard distributes the Year 1 revenue target across per-line monthlies (scaled by prior-year line share + line-specific seasonality), locks completed-month YTD actuals to the cent, and gracefully degrades when target is 0 or no prior-year line items exist.

Today the `byLine.length > 0` branch of `initializeFromXero` (`useForecastWizard.ts:763-770`) copies prior-year monthly values verbatim and ignores `targetRevenue` entirely. The "correct" target-aware logic exists ONLY in the empty-byLine fallback. This plan extends that pattern per-line, ships a regression test suite that locks in the contract, and fixes 3 call sites in `ForecastWizardV4.tsx` that silently strip `revenue_lines` from the API response (TypeScript will not catch this — the field is optional).

Output: 1 new test file + 3 production files modified, ~80 LOC prod + ~80 LOC tests, atomic PR.
</objective>

### Math contract (per RESEARCH.md §6)

For each `priorYear.revenue.byLine[]` entry:

```
lineShareOfPriorYear = line.total / priorYear.revenue.total           (guard: priorYear.revenue.total > 0)
lineYearTarget       = targetRevenue * lineShareOfPriorYear
ytdLine              = currentYTD.revenue_lines.find(case-insensitive trimmed name match)
lineYtdTotal         = ytdLine ? sum(ytdLine.by_month) : 0
ytdMonthKeys         = ytdLine ? Object.keys(ytdLine.by_month) : []
remainingTarget      = max(0, lineYearTarget - lineYtdTotal)

For each target-FY month key m (after remapMonthKeysToForecastYear on line.byMonth):
  if m is a YTD month (i.e. ytdMonthKeys includes the calendar-equivalent):
    year1Monthly[m] = round(ytdLine.by_month[matchingKey])             // YTD lock
  else:
    futureMonthsTotal = sum(line.byMonth[priorEquivMonth] for all FUTURE m's)
    monthRatio = futureMonthsTotal > 0
                   ? line.byMonth[priorEquivMonth] / futureMonthsTotal
                   : 1 / numFutureMonths                                // equal-split fallback
    year1Monthly[m] = round(remainingTarget * monthRatio)

// Residue absorption — guarantees per-line annual sum == round(lineYearTarget)
residue = Math.round(lineYearTarget) - sum(year1Monthly values)
if (residue !== 0 && lastFutureMonthKey) year1Monthly[lastFutureMonthKey] += residue
```

YTD-only lines (no prior-year match): append as fresh `RevenueLine` with YTD months populated, future months = 0, contributing $0 to remaining-year target (per RESEARCH.md case c / FCST-04).
