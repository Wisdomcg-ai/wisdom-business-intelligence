# Phase 28.4 — Cashflow Statement (Actuals, AASB 107)

## Objective

Add a proper Cashflow Statement view that uses Xero actuals for past months and
applies AASB 107 / IAS 7 four-list classification. This is a read-only report
view separate from the Cashflow Forecast — it answers "what happened to cash
last month" rather than "what will happen to cash next month".

## Scope

### Four-list classification

Uses `cashflow_statement_classification` table created in 28.1.

Every balance sheet account must be classified as one of:
- **Operating** — AR, AP, Inventory, GST, Accruals, Prepayments
- **Investing** — Fixed assets, term deposits, long-term investments
- **Financing** — Business loans, director loans, equity accounts
- **Non Cash** — Depreciation, Amortisation, FX differences

Sign convention (critical — AASB 107 compliant):
- Operating: Asset ↑ = outflow, Liability ↑ = inflow
- Investing: Asset purchase = outflow, asset sale = inflow
- Financing: Loan drawdown = inflow, repayment = outflow, dividend = outflow
- Non Cash: Always positive add-back to Net Profit

### Auto-classification logic

Based on `xero_accounts.xero_type`:
```ts
function autoClassify(account: XeroAccount): ListType {
  const t = account.xero_type
  if (t === 'BANK') return 'Unassigned' // Banks don't appear on statement
  if (t === 'CURRENT' || t === 'CURRLIAB') return 'Operating'
  if (t === 'FIXED' || t === 'INVENTORY' || t === 'NONCURRENT') return 'Investing'
  if (t === 'LIABILITY' || t === 'TERMLIAB') return 'Financing'
  if (t === 'EQUITY') return 'Financing'
  if (t === 'DEPRECIATION' || t === 'AMORTISATION') return 'NonCash'
  return 'Unassigned'
}
```

Coach can override via UI.

### Statement calculation engine

**`src/lib/cashflow/statement.ts`**
```ts
export function buildCashflowStatement(
  forecastId: string,
  period: { from: string, to: string },
  plLines: PLLine[],
  balanceSheetByMonth: Record<string, BalanceSheetSnapshot>,
  classifications: StatementClassification[],
  settings: CashflowSettings
): CashflowStatement {
  // 1. Compute Net Profit from P&L (Revenue - COGS - OpEx) for period
  const netProfit = computeNetProfit(plLines, period)

  // 2. Add back NonCash items (depreciation, amortisation)
  const depreciationAddback = sumNonCashItems(plLines, classifications)

  // 3. Operating adjustments: movement in Operating BS items
  const operatingAdjustments = computeOperatingMovements(
    balanceSheetByMonth,
    classifications.filter(c => c.list_type === 'Operating')
  )

  // 4. Investing adjustments: movement in Investing BS items
  const investingActivities = computeMovements(
    balanceSheetByMonth,
    classifications.filter(c => c.list_type === 'Investing')
  )

  // 5. Financing adjustments
  const financingActivities = computeMovements(
    balanceSheetByMonth,
    classifications.filter(c => c.list_type === 'Financing')
  )

  // 6. Opening + Closing cash (reconciliation check)
  const openingCash = getBankTotal(balanceSheetByMonth[period.from])
  const closingCash = getBankTotal(balanceSheetByMonth[period.to])
  const netChange = closingCash - openingCash

  return {
    period,
    net_profit: netProfit,
    noncash_addbacks: depreciationAddback,
    operating_activities: operatingAdjustments,
    investing_activities: investingActivities,
    financing_activities: financingActivities,
    net_cash_from_operating: netProfit + depreciationAddback + sumByType(operatingAdjustments, 'Operating'),
    net_cash_from_investing: sumByType(investingActivities, 'Investing'),
    net_cash_from_financing: sumByType(financingActivities, 'Financing'),
    net_change_in_cash: netChange,
    opening_cash: openingCash,
    closing_cash: closingCash,
    reconciles: Math.abs((netProfit + depreciationAddback + ...) - netChange) < 0.01
  }
}
```

### New API route

**`GET /api/forecast/cashflow/statement?forecast_id=xxx&from=yyyy-mm&to=yyyy-mm`**
- Calls `buildCashflowStatement` with all required data
- Returns structured statement

### New hook

**`useCashflowStatement(forecastId)`**
- Takes period parameters
- Loads all dependencies in parallel (plLines, balance sheets, classifications, settings)
- Returns `statement` object for UI

## Non-goals

- PDF export of the statement (can be added in a later phase if requested)
- Direct method statement (starting from cash receipts) — indirect is the standard
- Comparative period columns (just current period for now)

## Success criteria

1. Coach can classify balance sheet accounts into 4 lists
2. Auto-classify populates sensible defaults
3. Statement view reconciles: Net change in cash = Closing cash - Opening cash
4. All three sections (Operating/Investing/Financing) sum correctly
5. Non-cash items (depreciation) shown as add-back, not cash outflow
6. Clear warning if classifications incomplete
7. 6 new engine tests added and passing
