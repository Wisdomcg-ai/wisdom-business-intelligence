## What shipped

### AASB 107 Cashflow Statement

A proper Cashflow Statement view that uses Xero actuals for past months and
applies four-list classification (Operating / Investing / Financing / NonCash).
This is read-only and complements the existing forward-looking Forecast tab —
answers "what happened to cash" vs "what will happen".

### Core library

**`src/lib/cashflow/statement.ts`**
- `buildCashflowStatement()` — AASB 107 indirect-method builder
- Takes P&L net profit + depreciation add-back + balance sheet snapshots + classifications
- Produces structured statement with Operating / Investing / Financing sections
- Applies correct sign convention:
  - Asset ↑ in Operating/Investing → outflow
  - Liability ↑ in Operating/Financing → inflow
  - Equity ↑ in Financing → inflow
- Computes `reconciles` flag (net change in cash = op + inv + fin)
- Counts `unassigned_accounts` for UI warning
- `autoClassify(xero_type)` heuristic: BANK→Unassigned, CURRENT→Operating,
  FIXED→Investing, TERMLIAB/EQUITY→Financing, DEPRECIATN→NonCash

### API

**`GET /api/forecast/cashflow/statement?forecast_id=xxx&from=YYYY-MM&to=YYYY-MM`**
- Pulls P&L totals from `xero_pl_lines` for net profit + depreciation
- Fetches Xero balance sheet with `periods=N&timeframe=MONTH` for snapshots
- Maps account_name → xero_account_id via `xero_accounts` cache
- Loads classifications from `cashflow_statement_classification`
- Calls `buildCashflowStatement()` and returns the full statement

**`GET/POST /api/forecast/cashflow/classifications`**
- GET returns all classifications for a forecast
- POST with `auto_classify: true` seeds from `xero_accounts` using heuristic
- POST with single row upserts a classification

### Hook

**`useCashflowStatement(forecastId)`**
- Loads classifications on mount
- Exposes `loadStatement(from, to)` for on-demand fetch
- `autoClassify()` and `upsertClassification()` helpers
- Returns `statement`, `classifications`, loading states, error

### UI

**`StatementClassificationEditor.tsx`**
- Shows 5-way list count summary (Operating: 12, Investing: 3, etc.)
- Warning when any accounts are Unassigned
- "Auto-classify from Xero types" button
- Filterable row list with dropdown per row to reassign
- Color-coded by list type (blue=Operating, purple=Investing, green=Financing, grey=NonCash, amber=Unassigned)

**`CashflowStatementTab.tsx`**
- AASB 107 formatted layout with section headings
- Reconciliation badge: green checkmark if reconciles, amber warning if not
- Each section:
  - Operating: Net Profit + Depreciation + Working capital movements → Subtotal
  - Investing: Line items → Subtotal
  - Financing: Line items → Subtotal
  - Net change / Opening / Closing
- Classification editor embedded (collapsible)
- Auto-loads statement for the forecast's actuals period

**`/finances/cashflow/page.tsx`** — modified
- New tab switcher: **Forecast** vs **Statement (AASB 107)**
- Clean separation: forward-looking vs backward-looking views
