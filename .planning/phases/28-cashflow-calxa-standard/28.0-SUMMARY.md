## Bonus bug fix discovered during verification

While testing Phase 28.0, the cashflow data quality panel flagged **"No Xero
actuals loaded"** for the test client. Debugging this uncovered a **long-standing
silent bug** in `/api/monthly-report/sync-xero`:

- FK constraint: `xero_pl_lines.business_id ‚Üí businesses.id`
- Code was inserting: `business_profiles.id` (via `ids.profileId`)
- Every sync failed with FK violation `error 23503`
- The Vercel function then timed out trying to log the error (exceeded 60s on Hobby plan due to 24-month sync + 500ms delays between calls)
- Function was killed ‚Üí browser saw 500, no error ever reached our catch block

**Impact:** `xero_pl_lines` had been **silently empty** for affected clients
for an unknown period. Reports were falling back to `forecast_pl_lines.actual_months`
instead of fresh Xero data.

**Fixes applied:**
1. `sync-xero/route.ts:294` ‚Äî use `ids.bizId` instead of `ids.profileId`
2. Reduced sync from 24 ‚Üí 13 months (fits within Vercel 60s limit)
3. Added stage-tracking + debug_log table for future diagnosis
4. Added `console.trace` in `useXeroConnection.handleSync` for call-site tracing

Commit: `91e10dc`

After fix, verified: **47 Xero accounts synced** for the test client, data
quality panel confirms `‚úì 47 Xero accounts (actuals)`.
