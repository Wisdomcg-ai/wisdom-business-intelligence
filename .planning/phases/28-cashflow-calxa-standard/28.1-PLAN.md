# Phase 28.1 â€” Settings Foundation

## Objective

Replace fragile keyword-based account matching with explicit account ID selection.
Coach picks each important account from a Xero Chart of Accounts dropdown.
Existing keyword matching stays in place as fallback so clients without
configured settings continue to work.

## Scope

### New database tables

**Migration**: `supabase/migrations/20260418_cashflow_calxa_tables.sql`

```sql
-- Per-forecast cashflow settings (replaces assumptions.cashflow over time)
CREATE TABLE cashflow_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  forecast_id UUID NOT NULL REFERENCES financial_forecasts(id) ON DELETE CASCADE,
  business_id UUID NOT NULL,

  -- Bank & equity
  bank_account_ids JSONB DEFAULT '[]',              -- Xero account UUIDs (multi-select)
  retained_earnings_account_id TEXT,                 -- Xero account UUID
  current_year_earnings_account_id TEXT,

  -- GST
  gst_method TEXT CHECK (gst_method IN ('Cash','Accrual')) DEFAULT 'Accrual',
  gst_rate NUMERIC(5,4) DEFAULT 0.10,
  gst_collected_account_id TEXT,
  gst_paid_account_id TEXT,
  gst_schedule TEXT DEFAULT 'quarterly_bas_au',      -- named BasePeriods (seeded in 28.3)

  -- Wages / PAYG WH
  wages_expense_account_id TEXT,
  payg_wh_rate NUMERIC(5,4),
  payg_wh_liability_account_id TEXT,
  payg_wh_schedule TEXT DEFAULT 'quarterly_bas_au',

  -- Super
  super_expense_account_id TEXT,
  super_payable_account_id TEXT,
  super_rate NUMERIC(5,4) DEFAULT 0.115,
  super_schedule TEXT DEFAULT 'quarterly_super_au',

  -- Depreciation
  depreciation_expense_account_id TEXT,
  depreciation_accumulated_account_id TEXT,

  -- Debtors / Creditors (global DSO/DPO)
  debtors_account_id TEXT,
  creditors_account_id TEXT,

  -- Company Tax
  company_tax_rate NUMERIC(5,4) DEFAULT 0.25,
  company_tax_liability_account_id TEXT,
  company_tax_schedule TEXT DEFAULT 'quarterly_payg_instalment',

  -- Feature flag: use this settings row vs fall back to assumptions.cashflow
  use_explicit_accounts BOOLEAN DEFAULT false,

  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (forecast_id)
);

-- Per-account cashflow profile overrides (the Calxa Advanced view â€” used in 28.3)
CREATE TABLE cashflow_account_profiles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  forecast_id UUID NOT NULL REFERENCES financial_forecasts(id) ON DELETE CASCADE,
  xero_account_id TEXT NOT NULL,
  account_code TEXT,
  account_name TEXT,
  cashflow_type INTEGER CHECK (cashflow_type BETWEEN 1 AND 5),  -- Type 1-5 enum (28.3)
  days DOUBLE PRECISION,                                        -- Type 3 or 4
  distribution JSONB,                                           -- number[12], Type 2
  schedule_base_periods JSONB,                                  -- number[12], Type 5
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (forecast_id, xero_account_id)
);

-- Four-list classification for Cashflow Statement (28.4)
CREATE TABLE cashflow_statement_classification (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  forecast_id UUID NOT NULL REFERENCES financial_forecasts(id) ON DELETE CASCADE,
  xero_account_id TEXT NOT NULL,
  account_code TEXT,
  account_name TEXT,
  account_type TEXT,                                            -- Asset | Liability | Equity
  list_type TEXT CHECK (list_type IN ('Operating','Investing','Financing','NonCash','Unassigned')) DEFAULT 'Unassigned',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (forecast_id, xero_account_id)
);

-- Full Chart of Accounts cache (deduplicated across business, refreshed from Xero on demand)
CREATE TABLE xero_accounts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID NOT NULL,
  xero_account_id TEXT NOT NULL,
  account_code TEXT,
  account_name TEXT NOT NULL,
  xero_type TEXT,          -- BANK | CURRENT | CURRLIAB | FIXED | EXPENSE | REVENUE etc.
  xero_class TEXT,         -- ASSET | LIABILITY | EQUITY | REVENUE | EXPENSE
  xero_status TEXT,        -- ACTIVE | ARCHIVED
  tax_type TEXT,           -- BAS Excluded, GST on Expenses, etc.
  last_synced_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (business_id, xero_account_id)
);

-- RLS for all 4 tables (owner + coach + service_role pattern from report_templates)
```

### New API routes

1. **`GET/POST /api/forecast/cashflow/settings?forecast_id=xxx`**
   - GET returns `cashflow_settings` row or defaults
   - POST upserts with validation

2. **`GET/POST /api/Xero/chart-of-accounts-full?business_id=xxx`**
   - GET reads cached `xero_accounts` (or triggers refresh if stale >24h)
   - POST forces a fresh fetch from Xero `/Accounts` endpoint
   - Stores full COA with type/class/status

### New React hook

**`useCashflowSettings(forecastId)`** â€” parallel to `useCashflowForecast`
- Loads from `/api/forecast/cashflow/settings`
- Exposes `settings`, `updateSetting`, `save`, `isLoading`, `isSaving`
- Default `use_explicit_accounts: false` so existing behaviour preserved

**`useXeroAccounts(businessId)`** â€” COA picker data source
- Loads from `/api/Xero/chart-of-accounts-full`
- Groups by `xero_class` (Asset / Liability / Equity / Revenue / Expense)
- Filtered helpers: `bankAccounts`, `fixedAssetAccounts`, `liabilityAccounts`, etc.

### New UI component

**`CashflowAccountsPanel.tsx`** â€” new section in the Cashflow settings slide-out

Sections:
- **Bank Accounts** â€” multi-select from `xero_accounts` where `xero_type='BANK'`
- **Debtors & Creditors** â€” single-select (AR, AP) from current assets/liabilities
- **GST** â€” GST collected + GST paid accounts, rate, schedule dropdown
- **Wages & PAYG WH** â€” Wages expense + PAYG WH liability accounts, rate, schedule
- **Superannuation** â€” Super expense + Super payable accounts, rate, schedule
- **Depreciation** â€” Depreciation expense + Accumulated depreciation accounts
- **Company Tax** â€” Rate + liability account + schedule

Each dropdown shows `Code + Name` (e.g., "11200 Trade Debtors").

Auto-populate defaults where possible based on `xero_type`:
- First BANK account â†’ bank_account_ids
- First CURRENT with "Receivable"/"Debtor" in name â†’ debtors_account_id
- First CURRLIAB with "Payable"/"Creditor" â†’ creditors_account_id
- Account with "GST" or "BAS" in name â†’ gst_paid_account_id
- Account with "PAYG" â†’ payg_wh_liability_account_id
- Account with "Super" â†’ super_payable_account_id
- Account with "Depreciation" â†’ depreciation_expense_account_id

Toggle: **"Use explicit accounts"** â€” when off, engine uses keyword matching (current behaviour). When on, engine reads from `cashflow_settings`.

### Engine changes

Add helper functions in `engine.ts`:
```ts
function resolveAccount(settings: CashflowSettings | null, type: AccountType, fallbackLine: PLLine): boolean {
  if (!settings?.use_explicit_accounts) return keywordMatch(fallbackLine, type)
  return settings[`${type}_account_id`] === fallbackLine.xero_account_id
}
```

Used where engine currently keyword-matches (depreciation, employment, bank fees).
