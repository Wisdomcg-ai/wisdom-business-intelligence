## What shipped

### Data model

**Migration `20260419_cashflow_schedules.sql`**
- New table `cashflow_schedules` with `base_periods JSONB` (BasePeriods[12] array)
- RLS policies: system schedules readable by all; custom schedules owner/coach only
- Seeds 6 AU-standard system schedules:
  - `monthly` ‚Äî pays in same month
  - `quarterly_bas_au` ‚Äî Feb/Apr/Jul/Oct (AU BAS quarters)
  - `quarterly_super_au` ‚Äî Jan/Apr/Jul/Oct (Super payment months)
  - `quarterly_payg_instalment` ‚Äî same as BAS (Feb/Apr/Jul/Oct)
  - `quarterly_feb_may_aug_nov` ‚Äî alternate AU quarterly pattern
  - `annual_aug` ‚Äî single annual payment in August

The existing `cashflow_account_profiles` table (created in 28.1) becomes active
in this phase ‚Äî wired up via the new API + UI.

### Helper libraries

**`src/lib/cashflow/distributions.ts`**
- `daysToDistribution(days)` ‚Äî Calxa's days‚Üídistribution formula. Returns
  12-element array summing to exactly 100. Example: 45 days ‚Üí `[0, 50, 50, 0, ...]`
- `isValidDistribution(arr)` ‚Äî validates distribution array

**`src/lib/cashflow/schedules.ts`**
- `SYSTEM_SCHEDULES` ‚Äî in-memory copy of seeded schedules (fallback)
- `getPaymentMonth(accrualMonth, basePeriods)` ‚Äî lookup
- `isPaymentMonth(currentMonth, basePeriods)` ‚Äî check
- `resolveSystemSchedule(name)` ‚Äî get BasePeriods by name
- `isValidBasePeriods(arr)` ‚Äî validate BasePeriods[12]

### API

**`GET/POST /api/forecast/cashflow/profiles`**
- GET returns all `cashflow_account_profiles` rows for a forecast
- POST upserts a single profile (cashflow_type 1-5 + days/distribution/schedule)
- POST with `{ delete: true }` removes a profile
- Auth: owner or assigned coach

### UI

**`AccountProfileEditor.tsx`** ‚Äî new component
- Filterable account list (by code or name)
- Per-row Type 1-5 dropdown:
  - Type 1: Immediate (100% same month)
  - Type 2: Days Count Profile (computed, Phase 28.3+)
  - Type 3: Creditor Days override (input days)
  - Type 4: Debtor Days override (input days)
  - Type 5: Named Schedule (dropdown of 6 AU schedules)
- Save/Delete per row with toast feedback
- Profiles highlighted in brand-orange when set

**`CashflowAccountsPanel.tsx`** ‚Äî modified
- New "Advanced: Per-Account Timing" collapsible section at the bottom
- Only renders when `forecastId` is present
- Embeds `AccountProfileEditor` inside
