# Phase 28.2 — Algorithm Completeness

## Objective

Close the remaining algorithmic gaps so the engine matches Calxa's indirect-method
cashflow. Upgrades depreciation from keyword-based to explicit account IDs (now
available from 28.1), adds company tax and CapEx modules, and restructures
output rows to match Calxa's indirect-method layout.

## Scope

### Module 1: Depreciation (upgrade from interim keyword-match)

Interim behaviour from 28.0 used keyword matching. Now uses explicit account IDs
from `cashflow_settings.depreciation_expense_account_id`.

New helper: `src/lib/cashflow/depreciation.ts`
```ts
export function identifyDepreciationLines(
  plLines: PLLine[],
  settings: CashflowSettings | null
): PLLine[] {
  if (settings?.use_explicit_accounts && settings.depreciation_expense_account_id) {
    return plLines.filter(l => l.account_code === settings.depreciation_expense_account_id
                             || l.xero_account_id === settings.depreciation_expense_account_id)
  }
  // Fallback: keyword match
  return plLines.filter(l => {
    const n = l.account_name.toLowerCase()
    return n.includes('depreciation') || n.includes('amortisation') || n.includes('amortization')
  })
}

export function sumDepreciationForMonth(depnLines: PLLine[], monthKey: string, forecast: FinancialForecast): number {
  return depnLines.reduce((sum, l) => sum + Math.abs(getLineValue(l, monthKey, forecast)), 0)
}
```

In engine: at output row construction, add "Non-cash adjustments — Depreciation"
as a positive number above net movement (reducing the total expense outflows by
this amount — consistent with indirect method).

### Module 2: Company Tax

New helper: `src/lib/cashflow/company-tax.ts`
```ts
export function computeCompanyTax(
  months: string[],
  netProfitByMonth: Record<string, number>,
  settings: CashflowSettings
): Record<string, number> {
  // Estimate annual tax based on YTD net profit × rate
  // Pay on schedule (quarterly PAYG instalments, or annual lump sum)
  const taxByMonth: Record<string, number> = {}
  const annualNetProfit = Object.values(netProfitByMonth).reduce((a, b) => a + b, 0)
  const annualTax = annualNetProfit * settings.company_tax_rate

  // Distribute across payment months based on company_tax_schedule
  const schedule = resolveSchedule(settings.company_tax_schedule)
  const paymentMonths = months.filter(m => {
    const monthNum = parseInt(m.split('-')[1])
    return schedule.base_periods[monthNum - 1] === monthNum
  })

  const perPayment = annualTax / paymentMonths.length
  for (const m of paymentMonths) {
    taxByMonth[m] = perPayment
  }
  return taxByMonth
}
```

Used by engine to add a "Company Tax" line to `liability_lines` in payment months.

### Module 3: CapEx

New helper: `src/lib/cashflow/capex.ts`

**Challenge**: Xero doesn't put CapEx in the P&L — it's a balance sheet movement.
We need to detect asset purchases by fetching the balance sheet and computing
month-on-month changes in Fixed Asset accounts.

Approach:
1. Fetch Xero balance sheet with `periods=11&timeframe=MONTH` for the actuals period
2. Extract Fixed Asset accounts (identified by `xero_type='FIXED'` in `xero_accounts` table)
3. Compute movement per month: `fixedAssets[month] - fixedAssets[prevMonth]`
4. Positive movement = asset purchase = cash outflow in that month
5. For forecast months: read from `forecast_investments` table (existing but unused)

New API route: `GET /api/forecast/cashflow/capex?business_id=xxx&forecast_id=xxx`
- Returns `{ month: amount, ... }` for each month in forecast
- Engine adds these to `asset_lines` with negative value

### Module 5: Output restructure (indirect method)

Update `CashflowForecastMonth` type to include indirect-method sections:

```ts
interface CashflowForecastMonth {
  // ... existing fields

  // NEW: Indirect method sections
  net_profit: number           // Revenue - COGS - OpEx (accrual)
  depreciation_addback: number // Non-cash add-back
  debtor_adjustment: number    // Accrual-to-cash adjustment for revenue
  creditor_adjustment: number  // Accrual-to-cash adjustment for COGS
  gst_payment: number          // Net GST paid to ATO this month
  payg_wh_payment: number      // PAYG withheld paid to ATO this month
  super_payment: number        // Super paid this month
  company_tax_payment: number  // Company tax paid this month
  capex_movement: number       // Asset purchases this month
  loan_movement: number        // Loan principal + interest this month

  // Direct method sections retained for backwards compat
  // (bank_at_beginning, bank_at_end unchanged)
}
```

Update `CashflowForecastTable` to render both row layouts based on a toggle:
- "Direct" (current behaviour) — shows cash_inflows / cash_outflows / movement_in_*
- "Indirect" (new default when settings.use_explicit_accounts=true) — shows Net Profit → Depreciation → Debtor Adj → Creditor Adj → Tax Items → CapEx → Net Movement
