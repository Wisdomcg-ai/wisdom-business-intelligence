<objective>
Update the coach dashboard so per-client idea counts split into "owned by this client" vs "team-shared with this client (owned by a teammate)". The total stays identical to the pre-phase number — only its decomposition changes.

Purpose: With Phase 61 live, an idea that was previously invisible to the coach (because it wasn't owned by the client) can now appear in the client's view if a teammate shared it. The coach wants to see at-a-glance whether the client is generating ideas themselves or just receiving them. CONTEXT.md "Counts & dashboards" locks this requirement.

Output: Modified API route + paired tests + page render update. No new RPCs, no DB changes.
</objective>

<resolved_flags>
**Flag from PLAN-CHECK 61-06 → resolved here at user direction (2026-05-14):**

Coach total semantics: **PRESERVE TODAY'S HEADLINE COUNT.** The "ideas_total" number rendered on the coach dashboard must equal the pre-phase value (sum of all ideas in the client's business — i.e. the existing `business_id IN (...)` count). Sharing does not shrink the headline.

Resolution:
- API returns `{ ideas_total, ideas_private, ideas_team_shared, captured, under_review, approved, rejected, parked }`.
  - `ideas_total` = COUNT of all ideas where `business_id = client.business_id` (identical to today's behavior).
  - `ideas_private` = COUNT where `shared_with_all = false AND coalesce(array_length(shared_with, 1), 0) = 0`.
  - `ideas_team_shared` = `ideas_total - ideas_private` (or computed directly; both must agree).
- UI renders headline ("12 ideas") + a conditional sub-line *"X private · Y shared with team"* shown only when `ideas_team_shared > 0`. Single-user clients see no clutter.
- Existing `captured`/`under_review`/`approved`/`rejected`/`parked` aggregates are status filters — orthogonal to sharing — and stay unchanged.
- Regression test pins the headline-total value (e.g., asserts that a fixture with 12 ideas across mixed visibility returns `ideas_total: 12`).
</resolved_flags>

    ```typescript
    // Phase 61: extend ideas count into owned vs team-shared breakdown.
    // The pre-phase total is preserved; we only add decomposition.
    const ideasResp = await supabase
      .from('ideas')
      .select('id, user_id, shared_with_all, shared_with')
      .or(ownerOrBizFilter)   // unchanged filter
      // Optional refinement: filter out rows the client owner cannot see
      // (those would be teammate-owned with no shared_with_all and no client_owner_id in shared_with).
      // Use the SQL: shared_with_all = true OR user_id = <client_owner_id> OR <client_owner_id> = ANY(shared_with)

    if (ideasResp.error) {
      Sentry.captureException(ideasResp.error, {
        tags: { route: 'coach/client-completion' },
        extra: { context: '[coach/client-completion] ideas fetch failed', clientId: client.owner_id }
      } as any)
      // Fallback: keep pre-phase shape with zeros for the breakdown
      ideas_breakdown = { owned: 0, team_shared: 0, total: 0 }
    } else {
      const rows = ideasResp.data ?? []
      const ownerId = client.owner_id
      const owned = rows.filter(r => r.user_id === ownerId).length
      const team_shared = rows.filter(r =>
        r.user_id !== ownerId &&
        (r.shared_with_all === true || (r.shared_with ?? []).includes(ownerId))
      ).length
      ideas_breakdown = { owned, team_shared, total: owned + team_shared }
    }
    ```
