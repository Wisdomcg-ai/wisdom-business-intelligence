<objective>
Replace the strict per-user RLS on `daily_tasks` and broaden the SELECT path on `ideas` to honor the new sharing columns. Keep mutations owner-only. Add two SECURITY DEFINER RPCs (`mark_task_complete`, `mark_idea_status`) so recipients can flip status without an UPDATE policy carve-out.

Purpose: Asymmetric RLS is the load-bearing security boundary of this phase. RESEARCH.md §5 Risk 1 explicitly says: "Recommend Option B (dedicated RPC) unless the planner finds a reason to prefer A." This plan adopts Option B. Owner-only UPDATE means recipients literally cannot rename/archive/delete by any path — the only mutation channel for non-owners is the constrained RPC.

Output: One SQL migration file containing policy DROP/CREATE for both tables + two RPC functions with EXECUTE grants.
</objective>

    **Section C — status-flip RPCs (Option B from RESEARCH.md §5 Risk 1)**

    ```sql
    -- daily_tasks status flip for recipients (and owner)
    -- SECURITY DEFINER: the function runs with elevated privileges and performs its own
    -- visibility check, then narrowly updates only status + completed_at.
    -- It is the ONLY way a non-owner can mutate a daily_tasks row.
    CREATE OR REPLACE FUNCTION public.mark_task_complete(
      p_task_id uuid,
      p_completed boolean
    ) RETURNS public.daily_tasks
    LANGUAGE plpgsql
    SECURITY DEFINER
    SET search_path = public
    AS $$
    DECLARE
      v_task public.daily_tasks;
      v_uid uuid := auth.uid();
    BEGIN
      IF v_uid IS NULL THEN
        RAISE EXCEPTION 'Not authenticated' USING ERRCODE = '42501';
      END IF;

      -- Load + visibility check (same predicate as the SELECT policy)
      SELECT t.* INTO v_task
      FROM public.daily_tasks t
      WHERE t.id = p_task_id
        AND (
          t.user_id = v_uid
          OR (
            t.shared_with_all = true
            AND t.business_id IS NOT NULL
            AND t.business_id = ANY (public.auth_get_accessible_business_ids())
          )
          OR v_uid = ANY (t.shared_with)
        );

      IF NOT FOUND THEN
        RAISE EXCEPTION 'Task not found or access denied' USING ERRCODE = '42501';
      END IF;

      -- Narrow update: ONLY status + completed_at + updated_at.
      -- Do NOT touch user_id, business_id, shared_with_all, shared_with, title, etc.
      UPDATE public.daily_tasks
         SET status       = CASE WHEN p_completed THEN 'done' ELSE 'to-do' END,
             completed_at = CASE WHEN p_completed THEN now() ELSE NULL END,
             updated_at   = now()
       WHERE id = p_task_id
      RETURNING * INTO v_task;

      RETURN v_task;
    END;
    $$;

    REVOKE ALL ON FUNCTION public.mark_task_complete(uuid, boolean) FROM PUBLIC;
    GRANT EXECUTE ON FUNCTION public.mark_task_complete(uuid, boolean) TO authenticated;

      'Phase 61: lets a recipient (or owner) flip a shared daily_task between done/to-do. SECURITY DEFINER with manual visibility check + narrowed column update — bypasses the owner-only UPDATE policy by design.';
    ```
