<objective>
Create the public `/reports/view/[token]` route that the email's "View Report" button links to, and wire middleware so anonymous clients can actually reach it.

Two distinct middleware changes are required (RESEARCH.md §Pitfall 6 correction to D-23 which only mentioned one):
1. Add `/reports/view` to `publicRoutes` (line 86-96 of middleware.ts) so unauthenticated users are NOT redirected to `/auth/login`
2. Add `/reports/view` to `onboardingExemptRoutes` (line 142-151 of middleware.ts) so authenticated clients who haven't completed onboarding don't get redirected to `/business-profile`

Page renders from the frozen `snapshot_data` JSONB payload captured at approval time (D-15, D-18, D-19). Re-approval after an edit will update the snapshot — but the client's already-sent link continues to work because the same statusId resolves to whatever snapshot is currently stored. Any rendering-drift concerns are handled by the `schema_version` field in the payload (RESEARCH.md §Pitfall 3).

Purpose: The client clicks "View Report" in email -> sees exactly what Matt approved. No login. No Xero. No live data.
Output: One server page + one renderer client component + a 404 page + two middleware array edits.
</objective>

    ```typescript
    // Phase 35 D-19: Public read-only snapshot view rendered from cfo_report_status.snapshot_data.
    // Token signed with REPORT_LINK_SECRET (see src/lib/reports/report-token.ts).
    // This page is exempted from auth + onboarding checks by src/middleware.ts.
    import { notFound } from 'next/navigation'
    import { createClient } from '@supabase/supabase-js'
    import { verifyReportToken } from '@/lib/reports/report-token'
    import ReportSnapshotView from './ReportSnapshotView'

    export const dynamic = 'force-dynamic'
    export const runtime = 'nodejs'

    const supabase = createClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.SUPABASE_SERVICE_KEY!
    )

    interface PageProps {
      params: { token: string }
    }

    export default async function ReportViewPage({ params }: PageProps) {
      const statusId = verifyReportToken(params.token)
      if (!statusId) {
        notFound()
      }

      const { data: row, error } = await supabase
        .from('cfo_report_status')
        .select('id, business_id, period_month, snapshot_data, snapshot_taken_at, status')
        .eq('id', statusId)
        .maybeSingle()

      if (error) {
        console.error('[report-view] supabase error:', error)
        notFound()
      }
      if (!row || !row.snapshot_data) {
        notFound()
      }

      return (
        <main style={{ maxWidth: 960, margin: '0 auto', padding: '32px 16px' }}>
          <ReportSnapshotView
            snapshot={row.snapshot_data}
            snapshotTakenAt={row.snapshot_taken_at}
          />
        </main>
      )
    }
    ```

    **File B: `src/app/reports/view/[token]/ReportSnapshotView.tsx`** (client component that renders the payload):

    ```typescript
    'use client'
    // Read-only renderer for ReportSnapshotV1 payloads captured at approval time.
    // Keep it rendering-oriented: everything needed to render is already in the payload.
    // If schema_version mismatches what this file expects, render a graceful fallback
    // banner rather than crashing (RESEARCH.md §Pitfall 3).

    interface ReportSnapshotV1 {
      schema_version: number
      captured_at: string
      business: { id: string; name: string; slug: string | null; industry: string | null }
      period: { month: string; fiscal_year: number; label: string }
      coach: { name: string; email: string }
      report: unknown          // GeneratedReport — shape defined by Plan 35-06 caller
      commentary: unknown | null
      settings_applied: { sections: unknown; template_id: string | null }
      consolidated?: unknown
    }

    interface Props {
      snapshot: ReportSnapshotV1
      snapshotTakenAt: string | null
    }

    export default function ReportSnapshotView({ snapshot, snapshotTakenAt }: Props) {
      if (snapshot.schema_version !== 1) {
        return (
          <div style={{ padding: 24, background: '#fef3c7', borderRadius: 8 }}>
            <h1>Report snapshot captured in an older format</h1>
            <p>
              This link was generated before a platform update. Ask your coach to re-approve the
              report to regenerate this view.
            </p>
          </div>
        )
      }

      return (
        <article>
          <header style={{ marginBottom: 24 }}>
            <h1 style={{ margin: 0, fontSize: 28 }}>{snapshot.business.name}</h1>
            <p style={{ margin: '4px 0', color: '#6b7280' }}>{snapshot.period.label} financial report</p>
            {snapshotTakenAt && (
              <p style={{ margin: 0, color: '#9ca3af', fontSize: 13 }}>
                Captured {new Date(snapshotTakenAt).toLocaleDateString('en-AU', { day: 'numeric', month: 'long', year: 'numeric' })}
                {snapshot.coach?.name ? ` • Prepared by ${snapshot.coach.name}` : ''}
              </p>
            )}
          </header>

          {/*
            The detailed report body is rendered from snapshot.report (GeneratedReport shape).
            Plan 35-06 will refactor a read-only subset of the existing monthly-report page tabs
            to consume this payload. For Wave 2 we ship a minimal "report is ready" placeholder
            and populate the rich rendering in a follow-up task of Plan 35-06.
          */}
          <section style={{ padding: 24, border: '1px solid #e5e7eb', borderRadius: 8 }}>
            <p>Your {snapshot.period.label} financial report is available below.</p>
            <p style={{ color: '#6b7280', fontSize: 14 }}>
              The PDF attached to the email you received is the printable version of this report.
            </p>
          </section>
        </article>
      )
    }
    ```

    NOTE: This plan ships the minimum viable renderer. Plan 35-06 is responsible for fleshing out the rich tab-by-tab rendering. The placeholder above proves the flow end-to-end (token -> snapshot -> page) without blocking on the full read-only UI. Mark this explicitly in the file as a TODO-for-35-06.
