<objective>
Ship the on-demand "Import from Xero" UI on Step 4 of the forecast wizard. After this plan lands, the operator can:

1. Click an "Import from Xero" button in the Step 4 Team Members section header
2. See a modal listing every employee from the connected Xero tenant with checkboxes
3. Tick the ones to import → click "Import N selected" → those employees appear in the wizard with `payFrequency`, `standardHours`, `hourlyRate`, `currentSalary`, `_xeroFingerprint` all populated by the 52-00 helper
4. **Per Operator's Option D decision** — for hourly-imported employees, the annual-salary cell is **read-only by default with an "edit" affordance**; clicking edit switches to manual mode and marks the field as overridden. For salaried imports, the cell is editable by default; edits also flag overridden.
5. Operator-edited fields show a visual "edited" marker (small pill or distinctive border) so it's immediately obvious which fields the operator has overridden vs which are pure-Xero values.

**Operator's Option D is encoded throughout this plan:**
- Per-row provenance via `_xeroFingerprint` (set in 52-00)
- Per-field override tracking via `_overriddenFields: string[]` (allocated in 52-00, written here in 52-01)
- Read-only / editable branch on `_xeroFingerprint.calculationType`
- Edit affordance + visual marker

**Empty-state and rate-limit polish folds in here** (per 52-RESEARCH.md recommendation to drop standalone 52-03):
- Disabled button + tooltip for businesses without a Xero connection
- Inline error message on 429 / network failure / 0-employee response

**No re-import / no diff banner / no reconciliation in this plan.** Those land in 52-02. This plan is fresh imports only — every clicked employee is added as a new wizard row even if a row with the same name already exists. (52-02 introduces the `findMatchingTeamMember` matching path.)

**This plan does NOT touch rollup math.** Derived annual salary for hourly employees (`hourlyRate × standardHours × annualPayPeriods`) is a DISPLAY value that populates the `currentSalary` field on import; once written, it's just a number that flows through the existing rollup. ## New helpers in xero-payroll-mapping.ts (additive — types-only changes; existing exports untouched)

```typescript
import type { TeamMember, NewHire, PayFrequency } from '../types';

export const ANNUAL_PAY_PERIODS: Record<PayFrequency, number> = {
  weekly: 52,
  fortnightly: 26,
  monthly: 12,
};

/**
 * Derive annual salary for an hourly-imported employee.
 * Used for the read-only display when calculation_type === 'hourly'.
 * Returns undefined if any input is missing.
 */
export function getDerivedAnnualSalary(
  hourlyRate: number | undefined,
  standardHours: number | undefined,
  payFrequency: PayFrequency | undefined,
): number | undefined {
  if (hourlyRate == null || standardHours == null || !payFrequency) return undefined;
  const periods = ANNUAL_PAY_PERIODS[payFrequency];
  return Math.round(hourlyRate * standardHours * periods);
}

/**
 * Mark a field as operator-overridden. Returns the new _overriddenFields array
 * (existing fields preserved; idempotent on duplicate field names).
 */
export function markFieldOverridden(
  current: string[] | undefined,
  fieldName: keyof TeamMember | keyof NewHire | string,
): string[] {
  const existing = current ?? [];
  if (existing.includes(fieldName as string)) return existing;
  return [...existing, fieldName as string];
}

/**
 * Check whether a Xero-sourced field has been overridden by the operator.
 * Used for the visual 'edited' marker.
 */
export function isFieldOverridden(
  member: Pick<TeamMember, '_overriddenFields'>,
  fieldName: string,
): boolean {
  return Boolean(member._overriddenFields?.includes(fieldName));
}

/**
 * Decide whether a row is Xero-sourced (has provenance fingerprint).
 * Pure helper used for conditional rendering of the edit affordance + marker.
 */
export function isXeroSourcedRow(
  member: Pick<TeamMember, '_xeroFingerprint' | '_xeroEmployeeId'>,
): boolean {
  return Boolean(member._xeroEmployeeId);
}
```

## Modal component (inline in Step4Team.tsx — no separate file per 52-RESEARCH "no global Modal component" anti-pattern)

```tsx
const [showXeroImport, setShowXeroImport] = useState(false);
const [xeroEmployees, setXeroEmployees] = useState<XeroEmployeeApiShape[] | null>(null);
const [importLoading, setImportLoading] = useState(false);
const [importError, setImportError] = useState<string | null>(null);
const [hasXeroConnection, setHasXeroConnection] = useState<boolean | null>(null);  // null = unknown
const [selectedEmployeeIds, setSelectedEmployeeIds] = useState<Set<string>>(new Set());

// Probe connection on mount (cheap — same /api/Xero/employees endpoint, treats 404 as 'not connected')
useEffect(() => {
  let cancelled = false;
  (async () => {
    try {
      const res = await fetch(`/api/Xero/employees?business_id=${businessId}`, { method: 'HEAD' });
      // HEAD likely not supported; fall back to GET with a quick early-return on 404.
      // Actually: just rely on the GET response when the operator clicks. Set hasXeroConnection
      // optimistically to true here and update to false on first failed click.
      if (!cancelled) setHasXeroConnection(true);
    } catch {
      if (!cancelled) setHasXeroConnection(false);
    }
  })();
  return () => { cancelled = true; };
}, [businessId]);

const openXeroImport = async () => {
  setImportLoading(true);
  setImportError(null);
  setShowXeroImport(true);
  try {
    const res = await fetch(`/api/Xero/employees?business_id=${businessId}`);
    if (res.status === 404) {
      setHasXeroConnection(false);
      setImportError('Connect Xero to enable auto-import');
      setXeroEmployees([]);
      return;
    }
    const data = await res.json();
    if (data.expired || data.needs_reconnect) {
      setImportError(data.message || 'Reconnect Xero to access employee data');
      setXeroEmployees([]);
      return;
    }
    if (data.error) {
      // Friendly rate-limit handling — message comes through from existing route
      const isRateLimit = /rate limit|429|too many/i.test(String(data.error));
      setImportError(isRateLimit ? 'Xero rate limit hit — retry in a moment' : data.error);
      setXeroEmployees([]);
      return;
    }
    setXeroEmployees(data.employees ?? []);
  } catch (err) {
    setImportError(err instanceof Error ? err.message : 'Failed to fetch Xero employees');
    setXeroEmployees([]);
  } finally {
    setImportLoading(false);
  }
};

const importSelected = () => {
  if (!xeroEmployees) return;
  const today = new Date();
  const cutoff = new Date(today.getTime() + 7 * 24 * 3600 * 1000);  // today + 7 days
  for (const emp of xeroEmployees) {
    if (!selectedEmployeeIds.has(emp.employee_id)) continue;
    const enriched = enrichWizardMemberFromXeroEmployee(emp);
    const startDate = emp.start_date ? new Date(emp.start_date) : undefined;
    const isPlannedHire = startDate && startDate > cutoff;
    if (isPlannedHire) {
      const startMonth = `${startDate!.getFullYear()}-${String(startDate!.getMonth() + 1).padStart(2, '0')}`;
      actions.addNewHire({
        id: `hire-${Date.now()}-${emp.employee_id.slice(0, 6)}`,
        ...enriched,
        startMonth,
        salary: enriched.currentSalary ?? 0,
        superAmount: 0,
      } as any);  // type cast OK — all required NewHire fields set
    } else {
      actions.addTeamMember({
        id: `team-${Date.now()}-${emp.employee_id.slice(0, 6)}`,
        ...enriched,
        increasePct: 0,
        newSalary: enriched.currentSalary ?? 0,
        superAmount: 0,
      } as any);
    }
  }
  setShowXeroImport(false);
  setSelectedEmployeeIds(new Set());
};
```

## Annual salary cell read-only-vs-editable branch (in TeamTable row render)

```tsx
// In the existing TeamTable row, REPLACE the annual-salary input cell (currently
// always editable) with this conditional:
const xeroSourced = isXeroSourcedRow(member);
const wasHourlyImport = member._xeroFingerprint?.calculationType === 'hourly';
const overridden = isFieldOverridden(member, 'currentSalary');
const showReadOnly = xeroSourced && wasHourlyImport && !overridden;

{showReadOnly ? (
  <div className="flex items-center gap-1">
    <span className="text-sm">${member.currentSalary.toLocaleString()}</span>
    <button
      onClick={() => {
        // Switch to edit mode by marking the field as overridden — next render will show the input
        actions.updateTeamMember(member.id, {
          _overriddenFields: markFieldOverridden(member._overriddenFields, 'currentSalary'),
        });
      }}
      aria-label={`Edit annual salary for ${member.name}`}
      className="text-xs text-blue-600 underline"
    >
      Edit
    </button>
    <span className="text-xs text-gray-400" title="Derived from Xero hourly rate × standard hours × pay periods">
      (Xero)
    </span>
  </div>
) : (
  <>
    <input
      type="number"
      value={member.currentSalary}
      onChange={e => {
        const value = parseFloat(e.target.value) || 0;
        actions.updateTeamMember(member.id, {
          currentSalary: value,
          _overriddenFields: xeroSourced ? markFieldOverridden(member._overriddenFields, 'currentSalary') : member._overriddenFields,
        });
      }}
      aria-label={`Annual salary for ${member.name}`}
      className="..."
    />
    {xeroSourced && overridden && (
      <span className="ml-1 px-1.5 py-0.5 text-xs bg-amber-100 text-amber-700 rounded" aria-label="Edited from Xero value">edited</span>
    )}
  </>
)}
```
