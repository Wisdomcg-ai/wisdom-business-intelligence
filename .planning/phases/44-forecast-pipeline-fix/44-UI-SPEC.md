## Copywriting Contract

Phase 44 has no primary CTA, no destructive action, and no new error state — it is a passive informational change. Copy below is **binding and verbatim**; planner and executor must use these strings.

### Coverage banner (when `monthsCovered < expectedMonths`)

| Element | Copy | Notes |
|---------|------|-------|
| Eyebrow (uppercase) | `XERO DATA COVERAGE` | Mirrors `PlanPeriodBanner` eyebrow style |
| Main line — partial coverage | `Xero data covers {Mon YYYY} – {Mon YYYY} ({N} months)` | Example: `Xero data covers Mar 2025 – Apr 2026 (14 months)` (verbatim from CONTEXT D-15) |
| Main line — single month | `Xero data covers {Mon YYYY} only (1 month)` | Singular form for boundary case |
| Sub-line | `Forecast defaults are calculated from these months only. You can override any value below.` | Reassures coach the assumption is data-honest, not zero-padded |

**Banner is NOT shown** when `monthsCovered === expectedMonths` (full FY+prior FY = 24 months) — banner only renders for partial coverage. Always-on for sparse tenants; never dismissible (CONTEXT scope: "No — it should always show when coverage < full FY").

### Missing-month cell

| Element | Copy | Notes |
|---------|------|-------|
| Cell glyph | `—` (em-dash, U+2014) | Single character; alignment must match currency cell width |
| Tooltip on hover | `No data from Xero for this month.` | Verbatim from spawn prompt scope-constraint #2 |
| `aria-label` on cell | `No Xero data for {Mon YYYY}` | Screen-reader text; `{Mon YYYY}` resolved to e.g. "Mar 2025" |

### Real-zero cell

| Element | Copy | Notes |
|---------|------|-------|
| Cell glyph | `$0` (existing rendering — unchanged) | Distinct from `—`; means "Xero reported $0 for this account-month" |
| Tooltip | (no new tooltip; existing currency cell behavior preserved) | |

### Assumption-helper subtext (Step 3, possibly Step 4/5)

| Context | Copy template | Example |
|---------|---------------|---------|
| Average computed from data months only | `Based on {N} months of Xero data` | `Based on 14 months of Xero data` |
| Existing subtext for "based on prior year" | (unchanged) | (unchanged) |

**Binding rule:** the `{N}` is the count of real-data months actually used in the average — NOT `monthKeys.length`, NOT 12, NOT 24. Counter-pattern: `Based on 24 months of Xero data` when only 14 had data is a bug.

### Empty state (no Xero connection at all)

Out of scope for Phase 44 — existing wizard empty state is untouched. (Phase 44 only refines the partial-coverage path, not the no-connection path.)

### Error state

Out of scope for Phase 44 — sync errors are surfaced via existing toast (`sonner`) on the `XeroSyncButton` flow. No new error UI introduced by this phase.

### Destructive actions

None in Phase 44.

## Component Inventory (Phase 44 net-new)

Exactly one new component is introduced. Nothing else is added to the design system.

### `XeroCoverageBanner`

| Property | Value |
|----------|-------|
| Path | `src/app/finances/forecast/components/wizard-v4/components/XeroCoverageBanner.tsx` |
| Pattern source | `src/app/goals/components/PlanPeriodBanner.tsx` (Phase 43, commits c55e20e + 2acb2a7) |
| Mounting | `Step3RevenueCOGS.tsx` — top of component, above the existing tab/view-mode toggle, below the step header |
| Visibility rule | Render only when `coverage.monthsCovered < coverage.expectedMonths` (i.e. partial). For full coverage, banner is null. Never user-dismissible. |
| Surface | `bg-amber-50 border border-amber-200 rounded-lg p-4 mb-4` (verbatim from `PlanPeriodBanner.tsx:33`) |
| Layout | `flex items-start gap-3` — icon left, content stack right |
| Icon | `<Database className="w-5 h-5 text-amber-700 mt-0.5 flex-shrink-0" />` (lucide-react `Database` icon — semantically Xero/database; `Calendar` is reserved for plan-period banner) |
| Eyebrow | `text-xs font-bold uppercase tracking-wide text-amber-700 mb-1` |
| Main line | `text-sm font-semibold text-brand-navy` |
| Sub-line | `text-xs text-gray-600 mt-1 leading-relaxed` |
| Action button | **None** (banner is informational; CONTEXT scope: not dismissible, no Adjust action) |
| Props | `coverage: { firstPeriod: string; lastPeriod: string; monthsCovered: number; expectedMonths: number }` |

**Reasoning for `Database` icon (vs `Calendar`):** `PlanPeriodBanner` already owns the Calendar metaphor for "your plan dates". A separate icon prevents visual collision when both banners stack on the same page in adjacent flows. `Database` reads as "data state" which matches the banner's semantic load.

### Cell render adjustments (no new component — modify `Step3RevenueCOGS.tsx` cell render)

The existing monthly grid uses an `isActualMonth(monthKey)` check at line 33 to lock currency cells. Phase 44 adds a parallel `isMissingMonth(monthKey)` derived from the coverage record returned by `pl-summary` (or its successor `ForecastReadService.getMonthlyComposite`).

Render contract per cell:

```
if isMissingMonth(monthKey)        → render `—` glyph (text-gray-400, with tooltip + aria-label)
else if isActualMonth(monthKey)     → render existing locked-actual currency cell (unchanged)
else                                → render existing editable projected cell (unchanged)
```

Column width and row height MUST be identical across all three branches. This is a glyph swap, not a layout change.

## Accessibility & Interaction

| Concern | Spec |
|---------|------|
| Banner role | `role="status"` on the outer `<div>` so assistive tech announces the coverage statement on mount without interrupting focus |
| Banner contrast | amber-700 on amber-50 = 4.6:1 (passes WCAG AA for body text); brand-navy on amber-50 = 12.7:1 (passes AAA) |
| Missing cell tooltip | Use existing `Tooltip` primitive from `src/components/ui/Tooltip.tsx` — keyboard accessible (focus on cell shows tooltip), 200ms delay |
| Missing cell screen-reader | `aria-label="No Xero data for {Mon YYYY}"` — overrides the visual em-dash so screen readers announce semantically |
| Tab order | Banner is non-interactive — does NOT enter the tab order. Cells already have established tab order from existing implementation; missing cells remain reachable for screen readers but render `—` (no input). |
| Reduced-motion | No new animations introduced. Existing wizard animations unchanged. |
