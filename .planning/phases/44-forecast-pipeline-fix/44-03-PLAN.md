From `src/app/api/Xero/sync-all/route.ts:191-401` (current production code):
- Xero PL-by-Month response shape: `{ Reports: [{ Rows: [...]; Fields: [...] }] }`
- Top-level Rows include `Section` rows (Revenue / Less Cost of Sales / Operating Expenses / Net Profit) which contain nested `Rows`.
- Each `Section.Rows[]` Row has `Cells: [{Value: 'Account Name'}, {Value: '$amount'}, {Value: '$amount'}, ...]`
- The number of cells per row = 1 (account name) + N (months returned) — N can be < 12 for sparse tenants.
- Account_type is derived from the parent Section title:
  - "Revenue" / "Income" / "Trading Income" → 'revenue'
  - "Cost of Sales" / "Less Cost of Sales" / "COGS" → 'cogs'
  - "Operating Expenses" / "Less Operating Expenses" → 'opex'
  - "Other Income" → 'other_income'
  - "Other Expense" / "Less Other Expense" → 'other_expense'

The header row in the response (`Reports[0].Rows[0]` typically a `Header` Row) carries the period date strings (e.g. `"Jul-25"` or `"Jul 25"` or `"31 Jul 2025"`). Parser MUST handle all three formats Xero emits across tenant tiers.

Existing FY math from `src/lib/utils/fiscal-year-utils.ts`:
- `generateFiscalMonthKeys(fiscalYear: number, fyStartMonth: number = 7): string[]` — returns 12 keys "YYYY-MM" for that FY.
- `getCurrentFiscalYear(today: Date, fyStartMonth: number = 7): number`

Long-format row shape (must match the new `xero_pl_lines` schema in 44-02):
```typescript
type ParsedPLRow = {
  account_code: string | null;       // Xero AccountID — sometimes empty for header/total rows
  account_name: string;

  period_month: string;               // 'YYYY-MM-01' (matches Postgres date column)

};
```

Coverage record shape (D-10):

```typescript
  expected_months: number;    // typically 12 (one FY) or 24 (two FYs); orchestrator passes this in
};
```

Reconciliation result shape (D-08):
```typescript

  account_code: string | null;
  account_name: string;
  monthly_sum: number;
  fy_total: number;
  diff: number;
};
type ReconciliationResult = {
  status: 'ok' | 'mismatch';
  discrepancies: Discrepancy[];
  tolerance: number;          // the tolerance applied (e.g. 0.01)
};
```

Existing $-string parsing (from `sync-all/route.ts:212-219`):
```typescript
const parseAmount = (s: string | undefined | null): number => {
  if (!s) return 0;
  const trimmed = String(s).replace(/[\$,]/g, '').trim();
  if (trimmed === '' || trimmed === '-') return 0;
  // Accounting parens negative: ($1,234) → -1234
  const isParen = trimmed.startsWith('(') && trimmed.endsWith(')');
  const num = parseFloat(isParen ? trimmed.slice(1, -1) : trimmed);
  return isParen ? -Math.abs(num) : (Number.isNaN(num) ? 0 : num);
};
```
